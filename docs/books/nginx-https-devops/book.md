## Что читатель построит к концу книги

**Вариант A — Nginx (учебный):**
```
Интернет
    │  HTTPS (443)
    ▼
[ Nginx ] ← /etc/nginx/sites-available/myapp.conf
    │  HTTP (8000, внутренний)
    ▼
[ Python-приложение ] ← systemd сервис

[ ufw ] — закрыт всё, кроме 22/80/443
[ certbot ] — Let's Encrypt, автопродление
[ DNS ] — domain.ru → IP сервера
```

**Вариант B — Caddy (реальный сервер):**
```
Интернет
    │  HTTPS (443) — сертификат автоматически
    ▼
[ Caddy ] ← Caddyfile (3 строки)
    │  HTTP (8000, внутренний)
    ▼
[ Python-приложение ] ← systemd сервис или Docker

[ ufw ] — закрыт всё, кроме 22/80/443
[ DNS ] — domain.ru → IP сервера
```

## Как пользоваться книгой

1. **Читай по порядку** — главы построены друг на друге
2. **Смотри схемы** — ASCII-диаграммы показывают поток данных
3. **Делай упражнения** — теория без практики мертва
4. **Отмечай чеклисты** — в конце каждой главы
5. **Следи за порядком** — неправильная последовательность может отрезать от сервера

## Предварительные требования

- Пройден Модуль 1 (Linux для DevOps)
- Установленная Ubuntu Server (виртуалка или реальный сервер)
- Базовое умение: `systemctl`, `journalctl`, `nano`, `chmod`
