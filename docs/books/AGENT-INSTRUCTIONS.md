# Инструкция для ИИ-агента: Написание книги по Linux для DevOps

## Контекст проекта

Этот проект — обучение DevOps с нуля до самостоятельности.
Ученик — программист, который знает только 3 команды Linux.
Цель — научить его разворачивать Python-проекты на сервере с автодеплоем из Git.

**Первая книга** — это основа для Модуля 1: Linux для DevOps.

---

## Что за книга

**Название:** "Linux для DevOps: От новичка к уверенному пользователю"

**Целевая аудитория:**
- Программисты, которые хотят освоить Linux для DevOps
- Знают базовое понятие команд, но не уверены в себе
- Нужны практические навыки для работы с серверами

**Объём:** 150-200 страниц

**Стиль:**
- Простой, понятный язык без академизма
- Каждая команда объясняется **по одной** — не "стена текста"
- Много практики: после каждой главы — упражнения
- Реальные примеры из DevOps-практики
- Без воды, без лишних отступлений

## Структура книги

### Часть 1: Основы (Главы 1-5)

#### Глава 1: Что такое Linux и зачем он DevOps'у
- История Linux кратко (1-2 страницы, не больше)
- Дистрибутивы: почему Ubuntu для сервера
- Что такое терминал и чем он лучше GUI для DevOps
- Как устроен сервер: железо, ОС, сервисы
- Установка Ubuntu Server (пошаговая инструкция с скриншотами)

#### Глава 2: Первая команда и файловая система
- Что такое командная строка, промпт, курсор
- `pwd` — где я?
- `ls` — что здесь есть?
- `cd` — как перемещаться
- Абсолютные и относительные пути
- Файловая система Linux: `/`, `/home`, `/etc`, `/var`, `/usr`
- **Упражнения:** навигация по системе

#### Глава 3: Файлы и директории
- `mkdir` — создать папку
- `touch` — создать файл
- `cp`, `mv`, `rm` — копировать, переместить, удалить
- `cat`, `less`, `head`, `tail` — читать файлы
- `find` — искать файлы
- Скрытые файлы и `.` / `..`
- **Упражнения:** создать структуру проекта из командной строки

#### Глава 4: Права и владельцы
- `ls -l` — читать права
- `chmod` — менять права (цифры и буквы)
- `chown` — менять владельца
- `umask` — права по умолчанию
- `sudo` — зачем и когда использовать
- Пользователи и группы: `/etc/passwd`, `/etc/group`
- **Упражнения:** настроить права для веб-приложения

#### Глава 5: Редакторы в терминале
- nano: быстрый старт (основные хоткеи)
- vim: минимум для выживания (режимы, сохранение, выход)
- Когда какой использовать
- **Упражнения:** отредактировать конфиг, создать скрипт

### Часть 2: Процессы и сервисы (Главы 6-8)

#### Глава 6: Процессы
- Что такое процесс, PID, PPID
- `ps`, `ps aux` — посмотреть процессы
- `top`, `htop` — мониторинг в реальном времени
- `kill`, `killall` — остановить процесс
- `&`, `nohup`, `disown` — фоновые задачи
- `nice`, `renice` — приоритеты
- **Упражнения:** запустить, найти, остановить процесс

#### Глава 7: Системные сервисы и systemd
- Что такое демон/сервис
- `systemctl status/start/stop/restart/enable/disable`
- `journalctl` — смотреть логи сервиса
- Как написать свой сервис (`.service` файл)
- Автозапуск при загрузке
- **Упражнения:** создать свой сервис для Python-скрипта

#### Глава 8: Логи и диагностика
- Где лежат логи: `/var/log`
- `journalctl` — системные логи
- `dmesg` — сообщения ядра
- `tail -f` — следить за логами в реальном времени
- `grep` в логах — искать ошибки
- Ротация логогов: `logrotate`
- **Упражнения:** найти причину падения сервиса по логам

### Часть 3: Сеть и пользователи (Главы 9-11)

#### Глава 9: Пользователи и группы
- `useradd`, `usermod`, `userdel`
- `groupadd`, `groupmod`
- `/etc/shadow` — хэши паролей
- SSH-ключи: генерация, добавление на сервер
- `su` vs `sudo -i`
- Best practices: не работать под root
- **Упражнения:** создать пользователя для деплоя, настроить SSH

#### Глава 10: Сетевые основы
- IP-адреса, порты, протоколы (TCP/UDP)
- `ip addr` — посмотреть IP
- `ss` или `netstat` — открытые порты
- `ping` — проверка доступности
- `curl` — HTTP-запросы из терминала
- `wget` — скачать файл
- `nc` (netcat) — диагностика соединений
- DNS: `dig`, `nslookup`, `/etc/resolv.conf`
- **Упражнения:** диагностировать сетевые проблемы

#### Глава 11: Диски и файловые системы
- `df` — место на диске
- `du` — размер директорий
- `mount`, `umount` — монтирование
- `/etc/fstab` — автоматическое монтирование
- USB-диски и внешние хранилища
- **Упражнения:** найти что забивает диск, смонтировать диск

### Часть 4: Практика DevOps (Главы 12-14)

#### Глава 12: Shell-скрипты
- Shebang: `#!/bin/bash`
- Переменные, условия, циклы
- Аргументы скрипта (`$1`, `$2`, `$@`)
- Проверка ошибок (`set -e`)
- Вывод: stdout, stderr, перенаправление
- Скрипты для автоматизации рутины
- **Упражнения:** написать скрипт бэкапа, скрипт деплоя

#### Глава 13: Пакетные менеджеры и софт
- `apt update`, `apt upgrade`, `apt install`
- `snap` — альтернативная установка
- Поиск пакетов: `apt search`
- Удаление и очистка
- PPAs и сторонние репозитории
- **Упражнения:** установить и настроить стек LAMP

#### Глава 14: Итоговый проект
- Развернуть Python-приложение на сервере
- Настроить сервис через systemd
- Настроить логирование
- Настроить автозапуск
- Создать пользователя для приложения
- Настроить права
- Написать скрипты для управления
- **Чеклист:** всё работает после перезагрузки

## Принципы написания

### 1. Одна команда — одно объяснение
Не давай список из 10 команд подряд.
Каждую команду разбирай отдельно:
- **Что делает** (простыми словами)
- **Синтаксис** (шаблон использования)
- **Пример** (реальный, из практики DevOps)
- **Что может пойти не так** (частые ошибки)

### 2. Контекст важнее синтаксиса
Не просто `chmod 755 file`, а:
> "Когда ты загружаешь скрипт на сервер, он не запустится — у него нет прав на выполнение.
> `chmod +x script.sh` говорит Linux: 'разреши всем запускать этот файл'."

### 3. Упражнения после каждой главы
- 3-5 упражнений
- От простого к сложному
- Реальные задачи, а не абстрактные
- С ответами/подсказками в конце книги

### 4. Визуальные элементы
- **Совет** — для лучших практик
- **Опасно** — для команд, которые могут всё сломать
- **Запомни** — для ключевых концепций
- **Практика** — для упражнений

### 5. Никакой воды
- Без истории вида "когда я был молодой..."
- Без объяснения очевидного (что такое файл)
- Без параллельных тем (не про Docker в книге про Linux)

### 6. Реальные примеры
Все примеры из практики DevOps:
- Настройка веб-сервера
- Деплой приложения
- Диагностика проблем
- Автоматизация рутины

## Проверка качества

Перед сдачей каждой главы проверь:

1. **Понятность:** поймёт ли человек с 3 командами Linux?
2. **Практичность:** можно ли сразу применить?
3. **Полнота:** все ли команды объяснены?
4. **Безопасность:** есть ли предупреждения про опасные команды?
5. **Упражнения:** есть ли после каждой главы?
6. **Без воды:** нет ли лишних отступлений?
