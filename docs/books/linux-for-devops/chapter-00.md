> **Запомни:** Эта глава — не про Linux. Это про то как создать безопасную площадку где можно ломать без страха. Сделай всё по шагам — дальше будет только практика.

## 0.1 Что такое виртуальная машина

**Виртуальная машина (VM)** — это компьютер внутри компьютера.

```
Твой настоящий компьютер
└── VirtualBox (гипервизор)
    └── Ubuntu Server VM
        └── Linux внутри
```

**Зачем:**
- Можно ломать — ничего страшного
- Можно откатиться назад (снапшот)
- Изолировано от твоей основной системы
- Как настоящий сервер, но дома

**Что нужно:**
1. **VirtualBox** — бесплатная программа
2. **Ubuntu Server ISO** — образ установочного диска
3. **15 минут** — на настройку

## 0.2 Установка VirtualBox

### Windows

1. Скачай: https://www.virtualbox.org/wiki/Downloads
2. Нажми **"Windows hosts"**
3. Запусти скачанный `.exe`
4. Next → Next → Install
5. Если спросит про сетевые адаптеры — разреши

### macOS

1. Скачай: https://www.virtualbox.org/wiki/Downloads
2. Нажми **"macOS hosts"** (AMD64 для Intel, ARM64 для Apple Silicon)
3. Открой `.dmg` → запусти установщик
4. Разреши в Системных настройках если попросит

> **Apple Silicon (M1/M2/M3):** VirtualBox работает плохо.
> Лучше используй **UTM**: https://mac.getutm.app/
> Он бесплатный и работает нативно.

### Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install -y virtualbox virtualbox-ext-pack
```

Или скачай с сайта: https://www.virtualbox.org/wiki/Linux_Downloads

### Проверка установки

Открой VirtualBox. Увидишь пустое окно:

```
┌─────────────────────────────────┐
│  Oracle VM VirtualBox Manager   │
├─────────────────────────────────┤
│                                 │
│   Инструменты                    │
│                                 │
│   (пусто)                       │
│                                 │
│   [Создать]  [Настроить]        │
│                                 │
└─────────────────────────────────┘
```

Если открылось — всё ок. Идём дальше.

## 0.3 Скачивание Ubuntu Server

1. Открой: https://ubuntu.com/download/server
2. Найди **Ubuntu Server 24.04 LTS** (или последнюю LTS)
3. Нажми **Download**
4. Скачается `.iso` файл (~2.5 ГБ)

> **Запомни:** LTS = Long Term Support.
> Поддержка 5 лет. Это стандарт для серверов.
> Не бери не-LTS версии — они живут только 9 месяцев.

**Проверь что файл скачался:**
```bash
# На Windows (PowerShell):
ls ~\Downloads\*ubuntu*.iso

# На Mac/Linux:
ls ~/Downloads/*ubuntu*.iso
```

## 0.4 Создание виртуальной машины

### Шаг 1: Нажми «Создать»

В VirtualBox нажми кнопку **«Создать»** (или `Ctrl+N`).

### Шаг 2: Имя и тип

| Поле | Значение |
|------|----------|
| **Имя** | `devops-ubuntu` |
| **Папка** | Оставь по умолчанию |
| **Тип** | Linux |
| **Версия** | Ubuntu (64-bit) |

> **Если нет Ubuntu (64-bit):** Включи виртуализацию в BIOS.
> Windows: Диспетчер задач → Производительность → Виртуализация
> Если «Отключена» — перезагрузи → зайди в BIOS → включи.

### Шаг 3: Память (RAM)

**2048 МБ** (2 ГБ) — минимум для сервера.

| RAM компьютера | Сколько дать VM |
|----------------|-----------------|
| 4 ГБ | 2048 МБ |
| 8 ГБ | 4096 МБ |
| 16 ГБ+ | 4096 МБ |

> **Правило:** Не отдавай больше половины RAM компьютера.
> Иначе твоя основная система начнёт тормозить.

### Шаг 4: Процессор

**2 CPU** — если есть. Минимум 1.

### Шаг 5: Жёсткий диск

| Поле | Значение |
|------|----------|
| **Создать** | ✅ Создать новый виртуальный жёсткий диск |
| **Тип** | VDI (VirtualBox Disk Image) |
| **Формат** | Динамический (растёт по мере использования) |
| **Размер** | **25 ГБ** |

> **25 ГБ хватит** для обучения. Сервер без графики занимает ~3-4 ГБ.
> Динамический = на реальном диске займёт только то что используется.

### Шаг 6: Готово

Нажми **«Готово»**. VM появится в списке.

## 0.5 Подключение ISO-образа

VM создана, но пока пустая. Нужно подключить установочный диск.

1. Выбери машину `devops-ubuntu` → нажми **«Настроить»** (или `Ctrl+S`)
2. Перейди в раздел **«Носители»** (Storage)
3. Найди **«Контроллер: IDE»** → там будет пустой диск (значок CD с надписью «Пусто»)
4. Нажми на этот пустой диск
5. Справа нажми на **иконку диска** (рядом с «Оптический привод»)
6. Выбери **«Выбрать файл диска...»**
7. Найди скачанный `.iso` файл Ubuntu Server

Проверь что в списке носителей теперь:
```
Контроллер: IDE
  └─ [📀] ubuntu-24.04-live-server-amd64.iso
```

## 0.6 Настройка сети (важно!)

По умолчанию VM работает в **NAT** режиме. Интернет есть, но с твоего компьютера до VM не достучаться.

Для обучения это нормально. Но для SSH удобнее режим **«Сетевой мост»**.

### Вариант 1: NAT (по умолчанию) — ПРОЩЕ

✅ Интернет работает
✅ VM доступна изнутри
❌ С твоего компьютера не подключиться по SSH напрямую

**Для начала обучения — достаточно.**

### Вариант 2: Сетевой мост (Bridged) — УДОБНЕЕ

✅ Интернет работает
✅ VM доступна из твоей сети
✅ Можно подключиться по SSH с твоего компьютера
❌ Нужен доступ к роутеру (редко проблема)

### Как переключить на «Сетевой мост»

1. **«Настроить»** → **«Сеть»** (Network)
2. **«Тип подключения»** → выбери **«Сетевой мост»** (Bridged Adapter)
3. **«Имя»** → выбери свою сетевую карту (ту через которую подключён к интернету)
4. OK

> **Совет:** Для первых уроков оставь NAT. Когда освоишься — переключишь на мост для SSH.

## 0.7 Первый запуск

### Нажми «Запустить»

Зелёная стрелка → **Запустить**.

Откроется чёрное окно. Начнётся загрузка с ISO.

### Выбор языка

```
Welcome! Please select your language.
> English
  Русский
  Deutsch
  ...
```

**Выбери English.** Стрелками ↑↓ → Enter.

> **Почему не русский?** Серверы в 99% случаев на английском.
> Привыкай сейчас. Локализованные серверы — источник проблем.

### Раскладка клавиатуры

```
Keyboard configuration
Layout: [English (US)        ]
Variant: [English (US)       ]
```

Оставь **English (US)**. Done.

### Сеть

```
Network connections
eth0  DHCP  192.168.1.100  ✓
```

Если видит IP — интернет работает. **Continue.**

### Прокси

```
Configure proxy
Proxy address: [                    ]
```

**Оставь пустым.** Continue.

### Зеркало архива

```
Configure Ubuntu archive mirror
http://archive.ubuntu.com/ubuntu
```

Оставь по умолчанию. Ubuntu сама выберёт быстрое. Continue.

### Разметка диска

```
Storage configuration
[ ] Set up this disk as an LVM group
[x] Use an entire disk
```

Выбери **«Use an entire disk»**. Continue.

Подтверди: **Continue** (диск виртуальный, ничего не потеряешь).

### Профиль пользователя

```
Your name:     [Adelfos              ]
Server name:   [devops-ubuntu        ]
Username:      [adelfos              ]
Password:      [••••••••             ]
Confirm:       [••••••••             ]
```

| Поле | Что писать |
|------|-----------|
| **Your name** | Своё имя |
| **Server name** | `devops-ubuntu` (или любое) |
| **Username** | Простое имя латиницей |
| **Password** | **ЗАПОМНИ!** Запиши если нужно |

> **Запомни пароль!** Без него ты не войдёшь.
> На виртуалке нет «забыл пароль» кнопки.

### SSH Setup

```
SSH Setup
[ ] Install OpenSSH server
```

**Обязательно отметь!** ⬛ ✅

Без SSH ты не сможешь нормально работать с сервером.

### Snap-пакеты

```
Featured Server Snaps
[ ] certbot
[ ] nextcloud
[ ] ...
```

**Пропусти.** Continue.

Установишь потом через `apt`, когда понадобится.

### Установка

```
Installing system...
[████████████████░░░░] 80%
```

Займёт 5-10 минут. Можешь отвлечься.

### Перезагрузка

```
Installation complete!
[Reboot Now]
```

Нажми **Reboot Now**.

VM перезагрузится. Может попросить извлечь ISO — нажми Enter.

## 0.9 Снапшоты: машина времени

**Снапшот** — это «фотография» состояния VM. Можно откатиться к любому снапшоту.

> **Запомни:** Делай снапшот перед каждым экспериментом.
> Сломал? Откатился за 5 секунд.

### Создать снапшот

1. Выключи VM: `sudo poweroff`
2. В VirtualBox выбери машину → **Снимки** (Snapshots)
3. Нажми **«Сделать»** (камера)
4. Имя: `Clean Install`
5. Описание: `Свежая установка Ubuntu Server`

### Откатиться к снапшоту

1. Выбери машину → **Снимки**
2. Выбери снапшот → **«Восстановить»**
3. Подтверди

VM вернётся к состоянию на момент снапшота. Всё что было потом — потеряно.

> **Совет:** Хорошие моменты для снапшотов:
> - После чистой установки
> - Перед установкой Docker
> - Перед настройкой Nginx
> - Перед экспериментом с правами

## 0.10 Подключение по SSH

До сих пор ты работал в окне VirtualBox. Это неудобно — маленький экран, нет копирования, нельзя открыть два окна.

**SSH** (Secure Shell) — способ подключиться к серверу с твоего основного компьютера.

### Зачем SSH

| В окне VirtualBox | По SSH |
|-------------------|--------|
| Маленькое окно | Полный терминал |
| Нет копирования | Копируй как обычно |
| Одно окно | Сколько хочешь |
| Нужно переключаться | Работаешь на своём компьютере |

> **Запомни:** В реальной работе ты НИКОГДА не сидишь в консоли сервера.
> Только SSH.

### Узнай IP виртуалки

Внутри VM выполни:

```bash
hostname -I
192.168.1.100
```

Запомни этот IP. Он нужен для подключения.

> **Важно:** Если сеть в режиме NAT — SSH может не работать с твоего компьютера.
> Тогда переключи на **«Сетевой мост»** (раздел 0.6).

### Подключение по паролю

На своём компьютере (НЕ в VM) открой терминал:

```bash
# Windows PowerShell / Mac / Linux:
ssh adelfos@192.168.1.100
```

Замените `adelfos` на свой username, а IP — на IP своей VM.

Первый раз спросит:
```
The authenticity of host '192.168.1.100' can't be established.
Are you sure you want to continue connecting (yes/no)? yes
```

Набери `yes` → Enter.

Затем пароль от VM. Готово — ты на сервере!

```
Welcome to Ubuntu 24.04 LTS
...
adelfos@devops-ubuntu:~$
```

### Почему ключи лучше паролей

| Пароль | Ключ |
|--------|------|
| Можно подобрать | Практически невозможно |
| Нужно вводить каждый раз | Подключается автоматически |
| Можно забыть | Один раз настроил — работает вечно |

> **Совет:** Пароль — для первого раза. Ключи — для постоянной работы.
> С ключами пароль сервера больше не нужен.
> Если на ключе стоит passphrase — её может спросить SSH-клиент или `ssh-agent`.

### Что дальше

Подробно про SSH-ключи, алиасы и настройку `~/.ssh/config` — в Главе 9.
А пока достаточно знать: `ssh username@IP`.
