# Глава 5: Редакторы в терминале

> **Запомни:** На сервере нет мышки и графического редактора. Всё что ты хочешь изменить — через терминальный редактор. Научись хотя бы одному.

---

## 5.1 Зачем нужен терминальный редактор

На сервере ты будешь:
- Редактировать конфиги Nginx
- Менять настройки приложения
- Править скрипты
- Изменять переменные окружения

Всё это — текстовые файлы. Без графического интерфейса.

**Два основных выбора:**
1. **nano** — простой, интуитивный (начни с него)
2. **vim** — мощный, но сложный (выучи потом)

---

## 5.2 nano: быстрый старт

`nano` — самый простой редактор. Подсказки прямо внизу экрана.

### Открыть файл

```bash
nano ~/config.txt
```

Если файл не существует — создастся при сохранении.

### Интерфейс

```
  GNU nano 7.2                   /home/adelfos/config.txt
DATABASE_URL=postgres://localhost/mydb
SECRET_KEY=mysecret123

^G Help    ^O Write Out  ^W Where Is  ^K Cut    ^J Justify
^X Exit    ^R Read File  ^\ Replace   ^U Paste  ^_ Go to Line
```

Внизу — горячие клавиши. `^` значит `Ctrl`.

### Основные действия

| Действие | Клавиша | Что показывает |
|----------|---------|----------------|
| Сохранить | `Ctrl+O` | `^O Write Out` |
| Выйти | `Ctrl+X` | `^X Exit` |
| Поиск | `Ctrl+W` | `^W Where Is` |
| Вырезать строку | `Ctrl+K` | `^K Cut` |
| Вставить | `Ctrl+U` | `^U Paste` |

### Сохранить и выйти

1. `Ctrl+O` — записать (Write Out)
2. Enter — подтвердить имя файла
3. `Ctrl+X` — выйти

Или просто: `Ctrl+X`, потом `Y` (yes), потом Enter.

### Что может пойти не так

**Забыл сохранить и вышел:**
Nano спросит: `Save modified buffer? (Y)es (N)o`
- `Y` — сохранить
- `N` — выйти без сохранения
- `Ctrl+C` — отменить

## 5.3 nano: практический пример

Создай конфиг для приложения:

```bash
nano ~/myapp/.env
```

Введи:
```
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_NAME=myapp
SECRET_KEY=change_me_to_random
DEBUG=true
```

Сохрани: `Ctrl+O`, Enter, `Ctrl+X`.

Проверь:
```bash
cat ~/myapp/.env
```

> **Совет:** `.env` файлы — стандарт для хранения конфигурации приложений. Никогда не коммить их в Git!

## 5.4 vim: минимум для выживания

`vim` — мощный редактор. Но у него **модальная** система.

Это значит: одни и те же клавиши делают разное в разных режимах.

**Не пугайся.** Тебе нужно знать 5 команд для начала.

### Режимы vim

| Режим | Для чего | Как попасть |
|-------|----------|-------------|
| **Normal** | Навигация, команды | По умолчанию, `Esc` |
| **Insert** | Ввод текста | Нажми `i` |
| **Command** | Сохранить, выйти | Нажми `:` в Normal режиме |

### Самый важный навык: как выйти

```
1. Нажми Esc          (вернись в Normal режим)
2. Набери :q!         (выйти без сохранения)
3. Нажми Enter
```

Или для сохранения:
```
1. Нажми Esc
2. Набери :wq         (записать и выйти)
3. Нажми Enter
```

> **Запомни:** Если застрял — жми `Esc` потом `:q!` Enter. Это всегда работает.

### Минимальный набор команд

#### Открыть файл

```bash
vim ~/config.txt
```

#### Начать редактирование

Нажми `i` — перейдёшь в режим Insert (ввод текста).
Внизу появится `-- INSERT --`.

Теперь печатай как в обычном редакторе.

#### Перестать редактировать

Нажми `Esc` — вернёшься в Normal режим.
`-- INSERT --` исчезнет.

#### Сохранить

В Normal режиме (после `Esc`):
```
:w        ← запиши
```

#### Выйти

```
:q        ← выйти
:q!       ← выйти БЕЗ сохранения
:wq       ← сохранить и выйти
```

#### Отменить

В Normal режиме:
```
u         ← отменить последнее действие
Ctrl+r    ← повторить отменённое
```

#### Навигация (Normal режим)

| Клавиша | Действие |
|---------|----------|
| `h` `j` `k` `l` | Влево, вниз, вверх, вправо |
| `0` | В начало строки |
| `$` | В конец строки |
| `gg` | В начало файла |
| `G` | В конец файла |
| `/слово` | Поиск слова |
| `n` | Следующий результат |
| `dd` | Удалить строку |
| `yy` | Копировать строку |
| `p` | Вставить |

> **Совет:** Не пытайся выучить vim за день. Начни с: `i` → писать → `Esc` → `:wq`.
> Остальное выучишь когда понадобишься.

## 5.5 vim vs nano: когда какой использовать

| Ситуация | Выбор | Почему |
|----------|-------|--------|
| Быстро поправить конфиг | nano | Проще, быстрее |
| Написать скрипт | vim | Мощнее для кода |
| Ты на чужом сервере | nano | Он почти всегда есть |
| Ты на своём сервере | Что удобнее | Твой выбор |
| Нужно редактировать много | vim | Макросы, поиск/замена |
| Ты паникуешь | nano | Выйти проще |

> **Совет:** Выучи nano для быстрых правок. Изучай vim постепенно когда понадобишься больше.

## 5.6 Альтернатива: редактировать на своей машине

Есть лайфхак: редактировать файлы на сервере через SSH с своей машины.

### VS Code + Remote SSH

1. Установи VS Code
2. Установи расширение "Remote - SSH"
3. Подключись к серверу
4. Редактируй файлы как обычно

### SFTP клиенты

- **FileZilla** — графический, подключается по SSH
- **WinSCP** (Windows) — тоже графический

> **Совет:** Для обучения используй терминальные редакторы — это навык который останется с тобой даже когда нет GUI.

## 📝 Упражнения

### Упражнение 5.1: nano — создать и сохранить
**Задача:**
1. Открой nano: `nano ~/test.txt`
2. Напиши 3 строки текста
3. Сохрани: `Ctrl+O`, Enter
4. Выйди: `Ctrl+X`
5. Проверь: `cat ~/test.txt`

### Упражнение 5.2: nano — редактировать существующий
**Задача:**
1. Открой `.bashrc`: `nano ~/.bashrc`
2. Найди строку с `# some more ls aliases` (Ctrl+W)
3. Раскомментируй одну из строк (убери `#`)
4. Сохрани и выйди
5. Примени изменения: `source ~/.bashrc`

### Упражнение 5.3: vim — минимум
**Задача:**
1. Открой vim: `vim ~/vim-test.txt`
2. Нажми `i` — начни писать
3. Напиши "Hello from vim"
4. Нажми `Esc`
5. Набери `:wq` и Enter
6. Проверь: `cat ~/vim-test.txt`

### Упражнение 5.4: vim — навигация
**Задача:**
1. Создай файл с 10 строками:
   ```bash
   for i in {1..10}; do echo "Line $i" >> ~/lines.txt; done
   ```
2. Открой в vim: `vim ~/lines.txt`
3. Попробуй навигацию: `gg`, `G`, `j`, `k`, `/Line 5`
4. Удали строку: перейди на строку 5, нажми `dd`
5. Сохрани и выйди: `:wq`

### Упражнение 5.5: Создать конфиг приложения
**Задача:** Создай конфиг для Python-приложения:

```bash
nano ~/myapp/config.ini
```

Содержимое:
```ini
[server]
host = 0.0.0.0
port = 8000
debug = false

[database]
host = localhost
port = 5432
name = myapp
user = myapp_user
password = change_me

[logging]
level = INFO
file = /var/log/myapp/app.log
```

Сохрани и проверь через `cat`.
