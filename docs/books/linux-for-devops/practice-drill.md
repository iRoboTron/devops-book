# Тренировочный список — Книга 1: Linux для DevOps

> Выполняй задания по порядку. Команды не написаны — вспоминай сам.
> Если застрял — открой [practice-cheatsheet.md](practice-cheatsheet.md).
> В конце — одна команда которая удалит всё что ты наделал.

---

## Блок 1: Подключение и ориентация

1. Подключись по SSH к своему серверу (IP: твой, пользователь: твой)
2. Узнай в какой директории ты сейчас находишься
3. Посмотри содержимое текущей директории (обычный список)
4. Посмотри содержимое с подробностями: права, владелец, размер, дата
5. Посмотри содержимое включая скрытые файлы
6. Перейди в корень файловой системы
7. Посмотри что там есть
8. Вернись в домашнюю директорию одной командой
9. Узнай имя текущего пользователя
10. Узнай имя хоста (сервера)

---

## Блок 2: Файлы и директории

11. Создай директорию `~/devops-practice`
12. Перейди в неё
13. Создай внутри три директории: `app`, `logs`, `config`
14. Создай пустой файл `app/main.py`
15. Создай пустой файл `config/settings.conf`
16. Запиши в `config/settings.conf` строку: `PORT=8000`
17. Добавь ещё одну строку в тот же файл (не затирая первую): `DEBUG=true`
18. Выведи содержимое `config/settings.conf`
19. Скопируй `config/settings.conf` в `config/settings.conf.bak`
20. Переименуй `config/settings.conf.bak` в `config/settings.backup`
21. Выведи только первую строку файла `config/settings.conf`
22. Выведи только последнюю строку файла `config/settings.conf`
23. Найди все файлы в `~/devops-practice` с расширением `.py`
24. Найди все файлы в `~/devops-practice` с расширением `.conf`
25. Посмотри дерево директорий `~/devops-practice` (если установлен tree) или выведи список рекурсивно

---

## Блок 3: Права и владельцы

26. Посмотри права на файл `app/main.py`
27. Дай файлу `app/main.py` права: владелец читает+пишет+выполняет, остальные — только читают
28. Дай файлу `config/settings.conf` права: только владелец читает и пишет, остальные — ничего
29. Запиши в `app/main.py` строку: `#!/usr/bin/env python3`
30. Запусти `app/main.py` как исполняемый файл (должен выполниться без ошибок)
31. Посмотри от чьего имени ты выполняешь команды сейчас
32. Выполни команду `whoami` от имени root (через sudo)
33. Посмотри текущую umask

---

## Блок 4: Пользователи и SSH

34. Создай нового пользователя `devops`
35. Задай пользователю `devops` пароль
36. Добавь пользователя `devops` в группу `sudo`
37. Проверь в каких группах состоит пользователь `devops`
38. Создай директорию `~/.ssh` если её нет
39. Сгенерируй SSH-ключ (RSA, 4096 бит, без пароля, файл `~/.ssh/id_devops_practice`)
40. Посмотри публичный ключ который получился
41. Добавь публичный ключ в `~/.ssh/authorized_keys` пользователя `devops`
42. Установи правильные права на `~/.ssh/authorized_keys`
43. Посмотри список всех пользователей системы (первые 10 строк файла)
44. Заблокируй пользователя `devops` (чтобы не мог войти)
45. Разблокируй пользователя `devops`

---

## Блок 5: Процессы

46. Посмотри список всех запущенных процессов
47. Посмотри процессы в интерактивном режиме (выйди после просмотра)
48. Запусти в фоне: `sleep 300 &`
49. Посмотри список фоновых задач текущей сессии
50. Узнай PID процесса `sleep` который только что запустил
51. Завершни процесс `sleep 300` по PID (мягко)
52. Запусти `sleep 400` в фоне, найди его PID, убей принудительно
53. Посмотри сколько памяти используется на сервере (в удобном формате)
54. Посмотри сколько CPU используется прямо сейчас (одну строку статистики)
55. Найди все процессы с именем `bash`

---

## Блок 6: Системные сервисы

56. Создай файл юнита `/etc/systemd/system/practice-app.service` со следующим содержимым:
    - Description: Practice App
    - Тип: simple
    - Запускает: `/bin/sleep 9999`
    - Рестартует при падении
    - Запускается после network.target
57. Перезагрузи конфигурацию systemd (чтобы увидел новый файл)
58. Запусти сервис `practice-app`
59. Проверь статус сервиса `practice-app`
60. Включи сервис в автозапуск
61. Посмотри логи сервиса `practice-app` (последние 20 строк)
62. Останови сервис `practice-app`
63. Убедись что сервис остановлен
64. Отключи автозапуск сервиса `practice-app`

---

## Блок 7: Логи и диагностика

65. Посмотри последние 20 строк системного лога
66. Следи за системным логом в реальном времени (выйди через 5 секунд: Ctrl+C)
67. Найди в `/var/log/syslog` строки содержащие слово `error` (без учёта регистра)
68. Посмотри логи systemd за последние 10 минут
69. Посмотри логи только для сервиса `ssh`
70. Запиши в файл `~/devops-practice/logs/test.log` строку с датой:
    `echo "[$(date)] Test log entry" >> ~/devops-practice/logs/test.log`
71. Следи за этим файлом логов в реальном времени (в одном терминале)
72. В другой вкладке добавь ещё несколько строк в тот же файл
73. Найди в файле логов все строки за сегодняшнюю дату
74. Посчитай сколько строк в файле логов

---

## Блок 8: Сеть

75. Посмотри все сетевые интерфейсы и их IP-адреса
76. Посмотри какие порты слушают сервисы (только TCP, показывая процессы)
77. Проверь доступность google.com (3 пинга, потом стоп)
78. Скачай заголовки ответа с `http://example.com` (только заголовки, без тела)
79. Скачай содержимое `http://example.com` и сохрани в файл `~/devops-practice/example.html`
80. Посмотри первые 5 строк скачанного файла
81. Посмотри статус файрвола
82. Посмотри какие DNS-серверы настроены на сервере
83. Узнай внешний IP-адрес сервера (через curl к внешнему сервису)
84. Проверь доступность порта 80 на localhost

---

## Блок 9: Диски

85. Посмотри сколько места занято на всех дисках (в читаемом формате)
86. Посмотри размер директории `~/devops-practice`
87. Посмотри размер каждой поддиректории в `~/devops-practice`
88. Найди 5 самых больших файлов в `/var`
89. Посмотри список всех блочных устройств
90. Посмотри смонтированные файловые системы

---

## Блок 10: Пакеты

91. Обнови список пакетов
92. Найди пакет `htop` в репозитории
93. Установи `htop` (если не установлен)
94. Посмотри информацию о пакете `htop`
95. Установи `tree` (инструмент для просмотра дерева директорий)
96. Попробуй: `tree ~/devops-practice`
97. Проверь есть ли обновления для установленных пакетов
98. Удали `tree`

---

## Блок 11: Shell-скрипт

99. Создай файл `~/devops-practice/app/check.sh`
100. Напиши в нём скрипт который:
     - Выводит текущую дату и время
     - Выводит имя пользователя
     - Выводит свободное место на диске `/`
     - Если свободного места меньше 1GB — выводит "WARNING: low disk space"
     - Иначе выводит "OK: disk space is fine"
101. Сделай скрипт исполняемым
102. Запусти скрипт
103. Передай результат скрипта в `grep` и покажи только строки с датой

---

## Блок 12: Pipe и фильтры

104. Выведи список всех файлов в `/etc` и отфильтруй только те что содержат `nginx` в имени
105. Посмотри 10 самых нагруженных процессов по CPU (используй ps и sort)
106. Посчитай сколько файлов в директории `/etc`
107. Выведи уникальные расширения файлов в `/etc` (используй find, sed/awk, sort, uniq)
108. Найди все файлы в `/var/log` больше 1MB

---

## Финальная уборка

109. Останови и удали сервис `practice-app`:
     - Остановить сервис
     - Отключить автозапуск
     - Удалить файл юнита
     - Перезагрузить конфигурацию systemd

110. Удали пользователя `devops` вместе с его домашней директорией

111. Удали всё что создали в этой практике — **одна команда**:
     ```
     Удалить ~/devops-practice рекурсивно, без подтверждения
     ```

112. Проверь что папка удалилась
