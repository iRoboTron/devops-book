# Глава 11: Диски и файловые системы

> **Запомни:** Забитый диск = упавший сервер. Мониторить место и уметь работать с дисками — обязанность DevOps'а.

---

## 11.1 Сколько места: `df`

`df` = **d**isk **f**ree (свободное место)

```bash
df -h
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        25G   12G   12G  50% /
tmpfs           986M  1.2M  985M   1% /run
```

| Опция | Значение |
|-------|----------|
| `-h` | human-readable (ГБ, МБ вместо байт) |

### Разбор

| Колонка | Значение |
|---------|----------|
| `Filesystem` | Раздел/диск |
| `Size` | Общий размер |
| `Used` | Использовано |
| `Avail` | Доступно |
| `Use%` | Процент использования |
| `Mounted on` | Куда примонтирован |

> **Запомни:** Смотри `Use%`. Если больше 80% — время чистить.
> Если 95%+ — сервисы начнут падать.

### Только корневой раздел

```bash
df -h /
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        25G   12G   12G  50% /
```

## 11.2 Размер директорий: `du`

`du` = **d**isk **u**sage (использование диска)

### Размер текущей директории

```bash
du -sh
2.3G    .
```

| Опция | Значение |
|-------|----------|
| `-s` | summary (итог, без деталей) |
| `-h` | human-readable |

### Размер конкретной папки

```bash
du -sh /var/log
156M    /var/log
```

### Топ самых больших папок

```bash
du -sh /var/* | sort -rh | head -10
1.2G    /var/lib
456M    /var/log
234M    /var/cache
```

`sort -rh` = отсортировать по числам, в обратном порядке.
`head -10` = показать 10 самых больших.

> **Совет:** Когда диск забит — используй эту команду чтобы найти что занимает место:
> ```bash
> du -sh /* | sort -rh | head -10
> ```

## 11.3 Что такое монтирование

В Linux нет дисков `C:`, `D:`.
Всё находится внутри одного дерева от корня `/`.

**Монтирование** — это «подключение» диска к какой-то папке.

```
/
├── home/           ← на основном диске
├── var/            ← на основном диске
└── mnt/
    └── data/       ← здесь примонтирован второй диск
```

Когда ты пишешь в `/mnt/data/file.txt` — данные идут на второй диск.

## 11.4 Посмотреть диски: `lsblk`

```bash
lsblk
NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda      8:0    0   25G  0 disk
├─sda1   8:1    0   24G  0 part /
├─sda2   8:2    0    1K  0 part
└─sda5   8:5    0  976M  0 part [SWAP]
sdb      8:16   0  100G  0 disk
└─sdb1   8:17   0  100G  0 part /mnt/data
```

| Колонка | Значение |
|---------|----------|
| `NAME` | Имя устройства |
| `SIZE` | Размер |
| `TYPE` | disk (весь диск) или part (раздел) |
| `MOUNTPOINT` | Куда примонтирован |

> **Совет:** `lsblk` — самый наглядный способ посмотреть диски и разделы.

## 11.5 Смонтировать диск: `mount`

### Ручное монтирование

```bash
sudo mount /dev/sdb1 /mnt/data
```

Формат: `mount устройство точка_монтирования`

После этого содержимое `/dev/sdb1` появится в `/mnt/data`.

### Размонтировать

```bash
sudo umount /mnt/data
```

> **Важно:** Нельзя размонтировать если кто-то использует файлы внутри.
> Закрой все терминалы которые работают в этой директории.

### Проверить что смонтировано

```bash
mount | grep "^/dev"
/dev/sda1 on / type ext4 (rw,relatime,errors=remount-ro)
/dev/sdb1 on /mnt/data type ext4 (rw,relatime)
```

Или:
```bash
df -h
```

## 11.6 Автоматическое монтирование: `/etc/fstab`

`mount` работает до перезагрузки. Потом нужно монтировать снова.

**`/etc/fstab`** — файл который говорит что монтировать при загрузке.

```bash
cat /etc/fstab
# <filesystem>        <mount point>  <type>  <options>       <dump> <pass>
/dev/sda1             /              ext4    errors=remount-ro 0     1
/dev/sdb1             /mnt/data      ext4    defaults        0     2
```

### Разбор

| Поле | Значение |
|------|----------|
| `filesystem` | Устройство (или UUID) |
| `mount point` | Куда монтировать |
| `type` | Тип ФС (ext4, xfs, ntfs) |
| `options` | Опции монтирования |
| `dump` | 0 = не бэкапить |
| `pass` | 0 = не проверять, 1 = корень, 2 = остальные |

### Добавить запись

```bash
sudo nano /etc/fstab
# Добавить строку:
UUID=12345678-1234-1234-1234-123456789abc  /mnt/data  ext4  defaults  0  2
```

> **Совет:** Используй UUID вместо `/dev/sdb1` — имя может измениться.
> Узнать UUID: `blkid`

### Проверить fstab без перезагрузки

```bash
sudo mount -a
```

Примонтирует всё из fstab что ещё не примонтировано.
Если ошибка — скажет сразу.

> **Опасно:** Ошибка в fstab может привести к тому что сервер не загрузится.
> Всегда проверяй через `mount -a` перед перезагрузкой.

## 11.7 Типы файловых систем

| ФС | Где используется | Примечание |
|----|-----------------|------------|
| `ext4` | Linux (по умолчанию) | Стабильная, проверенная |
| `xfs` | Linux (серверы) | Быстрая для больших файлов |
| `btrfs` | Linux (новые) | Снапшоты, проверка целостности |
| `ntfs` | Windows | Чтение/запись из Linux |
| `fat32` | Флешки | Максимальный файл 4ГБ |
| `swap` | Файл подкачки | Не ФС, а область памяти |

> **Для DevOps:** ext4 — стандарт. Не заморачивайся если не нужно.

## 11.8 Найти что забивает диск

Диск забит на 95%. Что делать?

### Шаг 1: Общая картина

```bash
df -h
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        25G   24G  1G   96% /
```

### Шаг 2: Самые большие папки

```bash
sudo du -sh /* | sort -rh | head -5
12G     /var
8.2G    /home
2.1G    /usr
1.3G    /opt
```

### Шаг 3: Копать глубже

```bash
sudo du -sh /var/* | sort -rh | head -5
8.0G    /var/lib
4.2G    /var/log
1.1G    /var/cache
```

### Шаг 4: Ещё глубже

```bash
sudo du -sh /var/log/* | sort -rh | head -5
3.8G    /var/log/syslog
234M    /var/log/nginx
156M    /var/log/journal
```

### Шаг 5: Решение

```bash
# Очистить старые логи
sudo journalctl --vacuum-size=100M

# Очистить кэш пакетов
sudo apt clean

# Удалить старые ядра
sudo apt autoremove
```

> **Совет:** Сделай себе скрипт для мониторинга:
> ```bash
> #!/bin/bash
> echo "=== Disk Usage ==="
> df -h /
> echo ""
> echo "=== Top 5 directories ==="
> sudo du -sh /* 2>/dev/null | sort -rh | head -5
> ```

## 11.9 Файл подкачки: `swap`

**Swap** — это когда оперативная память кончилась и система использует диск.

Медленно. Но лучше чем упасть.

### Проверить swap

```bash
free -h
              total   used   free   shared  buff/cache  available
Mem:          1.9Gi   823Mi  412Mi   45Mi    746Mi       1.3Gi
Swap:         1.0Gi     0B   1.0Gi
```

### Создать swap файл

```bash
# Создать файл 2ГБ
sudo fallocate -l 2G /swapfile

# Или если fallocate нет:
sudo dd if=/dev/zero of=/swapfile bs=1M count=2048

# Правильные права
sudo chmod 600 /swapfile

# Сделать swap
sudo mkswap /swapfile

# Включить
sudo swapon /swapfile

# Проверить
free -h
```

### Добавить в fstab

```bash
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

> **Совет:** Swap = 2× RAM если RAM < 4ГБ.
> Swap = RAM если RAM 4-8ГБ.
> Swap не нужен если RAM > 16ГБ (но пусть будет 1-2ГБ на всякий случай).
