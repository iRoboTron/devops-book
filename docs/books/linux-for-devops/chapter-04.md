# Глава 4: Права и владельцы

> **Запомни:** Права доступа — это основа безопасности Linux. Без понимания прав ты не сможешь нормально настроить веб-сервис, деплой или скрипт.

---

## 4.1 Зачем нужны права

Linux — многопользовательская система. На одном сервере работают десятки программ и пользователей.

**Вопрос:** как защитить файлы одного пользователя от другого?

**Ответ:** права доступа.

Каждый файл имеет:
- **Владельца** — кто создал (или кому принадлежит)
- **Группу** — группа пользователей которым можно
- **Права** — что можно делать (читать, писать, выполнять)

---

## 4.2 Читаем права: `ls -l`

```bash
ls -l /home/adelfos
-rw-r--r-- 1 adelfos adelfos  220 апр  9 10:30 .bash_logout
-rw-r--r-- 1 adelfos adelfos 3771 апр  9 10:30 .bashrc
drwxr-xr-x 2 adelfos adelfos 4096 апр  9 10:30 projects
```

Посмотри на первую колонку: `-rw-r--r--`

Это **10 символов**. Разберём каждый.

### Первый символ — тип

| Символ | Значение |
|--------|----------|
| `-` | Обычный файл |
| `d` | Директория |
| `l` | Ссылка (link) |

### Остальные 9 символов — права

Разбиваются на 3 группы по 3 символа:

```
-  rwx  r-x  r-x
↑   ↑    ↑    ↑
|   |    |    └─ Остальные (others)
|   |    └────── Группа (group)
|   └─────────── Владелец (user/owner)
└─────────────── Тип файла
```

### Что значат буквы

| Буква | Значение |
|-------|----------|
| `r` | **Read** — читать |
| `w` | **Write** — писать/изменять |
| `x` | **Execute** — выполнять (для файлов) / заходить (для директорий) |
| `-` | Нет этого права |

### Разбираем пример: `-rw-r--r--`

```
-  rw-  r--  r--
↑   ↑    ↑    ↑
|   |    |    └─ Остальные: только чтение (r--)
|   |    └────── Группа: только чтение (r--)
|   └─────────── Владелец: чтение + запись (rw-)
└─────────────── Это файл (не директория)
```

**Перевод:** Владелец может читать и писать. Все остальные — только читать.

### Разбираем директорию: `drwxr-xr-x`

```
d  rwx  r-x  r-x
↑   ↑    ↑    ↑
|   |    |    └─ Остальные: чтение + заход (r-x)
|   |    └────── Группа: чтение + заход (r-x)
|   └─────────── Владелец: всё (rwx)
└─────────────── Это директория
```

**Перевод:** Владелец делает что хочет. Остальные могут смотреть содержимое и заходить.

> **Запомни:** Для директории `x` значит **заходить** в неё. Без `x` ты не можешь даже посмотреть файлы внутри.

## 4.3 Менять права: `chmod`

`chmod` = **ch**ange **mod**e (сменить режим прав)

### Два способа записи

1. **Буквенный** (понятный)
2. **Цифровой** (короткий)

---

### Буквенный способ

```
кто + или - что
```

| Кто | Буква |
|-----|-------|
| `u` | User (владелец) |
| `g` | Group (группа) |
| `o` | Others (остальные) |
| `a` | All (все) |

| Что | Буква |
|-----|-------|
| `r` | Read |
| `w` | Write |
| `x` | Execute |

**Примеры:**

```bash
chmod +x script.sh         # добавить выполнение ВСЕМ
chmod u+x script.sh        # добавить выполнение ВЛАДЕЛЬЦУ
chmod u+w file.txt         # разрешить владельцу писать
chmod o-r secret.txt       # убрать чтение у остальных
chmod a+x script.sh        # добавить выполнение всем (как +x)
chmod go-w file.txt        # убрать запись у группы и остальных
```

---

### Цифровой способ

Каждому праву соответствует число:

| Право | Число |
|-------|-------|
| `r` | 4 |
| `w` | 2 |
| `x` | 1 |
| `-` | 0 |

Складываем для каждой группы:

```
rwx = 4 + 2 + 1 = 7
rw- = 4 + 2 + 0 = 6
r-x = 4 + 0 + 1 = 5
r-- = 4 + 0 + 0 = 4
```

**Примеры:**

```bash
chmod 755 script.sh    # владелец: всё, остальные: чтение+выполнение
chmod 644 file.txt     # владелец: чтение+запись, остальные: чтение
chmod 600 secret.key   # только владелец читает и пишет
chmod 777 public/      # все делают всё (ОПАСНО!)
```

### Таблица частых прав

| Цифры | Файл | Директория |
|-------|------|------------|
| `777` | Все делают всё | Все делают всё |
| `755` | Владелец всё, остальные чтение+выполнение | Владелец всё, остальные заход+просмотр |
| `644` | Владелец чтение+запись, остальные чтение | — |
| `600` | Только владелец | — |
| `444` | Только чтение всем | — |

> **Запомни:**
> - Для скриптов: `755` (чтобы запускались)
> - Для конфигов: `644` (только читать и редактировать владельцу)
> - Для секретов (ключи, пароли): `600` (только владелец)
> - `777` — это зло. Никогда.

## 4.4 Менять владельца: `chown`

`chown` = **ch**ange **own**er (сменить владельца)

### Синтаксис

```bash
chown владелец:группа файл
```

### Поменять владельца

```bash
sudo chown nginx /var/www/html
```

Файл теперь принадлежит пользователю `nginx`.

### Поменять владельца и группу

```bash
sudo chown www-data:www-data /var/www/html
```

Файл принадлежит `www-data` и группе `www-data`.

### Поменять только группу

```bash
sudo chown :developers project/
```

`:` перед именем группы = менять только группу.

### Рекурсивно (все файлы внутри)

```bash
sudo chown -R www-data:www-data /var/www/html
```

`-R` = рекурсивно для всех файлов и поддиректорий.

> **Совет:** Когда разворачиваешь веб-приложение, ты почти всегда будешь делать:
> ```bash
> sudo chown -R www-data:www-data /var/www/myapp
> ```
> Это передаёт все файлы веб-серверу.

## 5.5 Права по умолчанию: `umask`

Когда ты создаёшь файл — у него уже есть какие-то права. Откуда?

**`umask`** определяет какие права **забрать** при создании.

```bash
umask
0022
```

`022` значит: забери запись у группы и остальных.

| Создаёшь | Без umask | С umask 022 | Итог |
|----------|-----------|-------------|------|
| Файл | 666 (rw-rw-rw-) | 644 (rw-r--r--) | Запись только владельцу |
| Директория | 777 (rwxrwxrwx) | 755 (rwxr-xr-x) | Запись только владельцу |

> **Запомни:** `umask` не добавляет права — он **забирает** права.
> 
> `umask 022` — стандартное значение. Оставь его в покое.

## 4.6 `sudo`: временный root

`sudo` = **s**uper**u**ser **do** (выполни от суперпользователя)

### Обычное использование

```bash
sudo apt update
```

Linux попросит пароль пользователя. Выполнит команду от root.

### Когда нужен `sudo`

| Действие | Нужен sudo? |
|----------|-------------|
| Работать в своей домашней папке | ❌ Нет |
| Устанавливать программы | ✅ Да |
| Редактировать файлы в `/etc` | ✅ Да |
| Перезапускать сервисы | ✅ Да |
| Смотреть чужие файлы | ✅ Да |
| Работать с дисками | ✅ Да |

### Правило

> **Правило:** Если команда обращается к системным файлам или сервисам — нужен `sudo`.
> Если работаешь в своём `~` — не нужен.

## 4.7 Реальный пример: права для веб-приложения

Представь: ты развернул Python-приложение в `/var/www/myapp`.

Nginx работает от пользователя `www-data`.

Твоё приложение должно:
- Читать файлы конфигурации
- Писать логи
- Загружать файлы пользователей

**Настройка прав:**

```bash
# 1. Передать директорию веб-серверу
sudo chown -R www-data:www-data /var/www/myapp

# 2. Директории — 755 (заходить и читать)
sudo chmod 755 /var/www/myapp
sudo chmod 755 /var/www/myapp/uploads

# 3. Конфиги — 644 (только чтение для www-data)
sudo chmod 644 /var/www/myapp/config.ini

# 4. Логи — 644 (чтение, владелец может писать)
sudo chmod 644 /var/www/myapp/app.log

# 5. Секреты — 600 (только владелец)
sudo chmod 600 /var/www/myapp/.env
```

> **Запомни:** Никогда не ставь `777` на директорию с загрузками.
> Если нужно чтобы веб-сервер писал — поставь `755` и убедись что владелец `www-data`.

## 4.8 Специальные права (кратко)

Есть три специальных бита. Ты не будешь их использовать каждый день, но должен знать что существуют.

### SUID (4xxx)

Файл запускается от **владельца файла**, а не от того кто запустил.

```bash
chmod 4755 /usr/bin/passwd
```

Команда `passwd` работает от root — чтобы обычный пользователь мог менять свой пароль.

> **Опасно:** SUID на неправильном файле = дыра в безопасности.

### SGID (2xxx)

Новые файлы в директории наследуют **группу** директории.

```bash
chmod 2775 /shared
```

Все кто создают файлы в `/shared` — файлы принадлежат группе директории.

### Sticky bit (1xxx)

В директории удалять файлы может **только владелец файла** (не директории).

```bash
chmod 1777 /tmp
```

`/tmp` имеет sticky bit — иначе любой мог бы удалить чужие временные файлы.

### Упражнение 4.2: Изменение прав

2. Поставь право на выполнение: `chmod +x ~/script.sh`
3. Проверь: `ls -l ~/script.sh` — появились ли `x`?
4. Убери право на выполнение: `chmod -x ~/script.sh`
5. Проверь снова

### Упражнение 4.3: Цифровые права
**Задача:**
1. Создай файл: `touch ~/secret.txt`
2. Поставь `600`: `chmod 600 ~/secret.txt`
3. Проверь: `ls -l ~/secret.txt` — только владелец?
4. Поставь `644`: `chmod 644 ~/secret.txt`
5. Теперь `755`: `chmod 755 ~/secret.txt`
6. Какой результат в каждом случае?

### Упражнение 4.4: Смена владельца
**Задача:**
1. Создай файл: `touch ~/owned.txt`
2. Попробуй сменить владельца: `sudo chown root ~/owned.txt`
3. Проверь: `ls -l ~/owned.txt` — кто теперь владелец?
4. Верни себе: `sudo chown $USER ~/owned.txt`
5. Проверь снова

### Упражнение 4.5: Практика для DevOps
**Задача:** Создай структуру веб-приложения и настрой права:

```bash
# 1. Создай структуру
mkdir -p ~/webapp/{public,logs,config}
touch ~/webapp/public/index.html
touch ~/webapp/logs/app.log
touch ~/webapp/config/settings.ini

# 2. Представь что ты настраиваешь для www-data
# (на виртуалке)
sudo chown -R www-data:www-data ~/webapp
sudo chmod 755 ~/webapp
sudo chmod 755 ~/webapp/public
sudo chmod 644 ~/webapp/public/index.html
sudo chmod 644 ~/webapp/logs/app.log
sudo chmod 640 ~/webapp/config/settings.ini

# 3. Проверь результат
ls -lR ~/webapp
```

Убедись что права соответствуют задуманному.
