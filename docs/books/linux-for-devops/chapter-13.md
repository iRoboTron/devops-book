# Глава 13: Пакетные менеджеры и софт

> **Запомни:** Устанавливать программы на сервер — базовый навык. Но важно не только установить, но и обновлять, и удалять.

---

## 13.1 Что такое пакетный менеджер

**Пакет** — это программа готовая к установке. Как `.exe` установщик но автоматизированный.

**Пакетный менеджер** — программа которая:
- Скачивает пакеты из репозитория
- Устанавливает их
- Обновляет
- Удаляет
- Следит за зависимостями

В Ubuntu это **apt** (Advanced Package Tool).

---

## 13.2 `apt`: установка программ

### Обновить список пакетов

```bash
sudo apt update
```

Это НЕ обновляет программы. Это обновляет **список** доступных программ.

> **Запомни:** Всегда делай `apt update` перед `apt install`.
> Иначе можешь устанавливать старую версию.

### Обновить все программы

```bash
sudo apt upgrade
```

Обновляет все установленные программы до последних версий.

### Обновить список и программы вместе

```bash
sudo apt update && sudo apt upgrade -y
```

`&&` = выполнить вторую если первая успешна.
`-y` = не спрашивать подтверждение.

> **Совет:** Делай это раз в неделю на сервере. Обновления = безопасность.

## 13.3 Установить программу

```bash
sudo apt install nginx
```

### Установить без подтверждения

```bash
sudo apt install -y nginx
```

### Установить несколько

```bash
sudo apt install -y nginx postgresql-14 redis-server
```

### apt сам разберётся с зависимостями

```bash
sudo apt install -y python3-pip
# pip установится вместе со всеми нужными библиотеками
```

> **Запомни:** apt сам скачает и установит зависимости.
> Не нужно вручную ставить библиотеки как в Windows.

## 13.4 Поиск пакетов

```bash
apt search nginx
```

Покажет все пакеты связанные с nginx.

### Узнать информацию о пакете

```bash
apt show nginx
Package: nginx
Version: 1.24.0-1
Description: small, powerful, scalable web/proxy server
...
```

---

## 13.5 Удалить программу

### Удалить но оставить конфиги

```bash
sudo apt remove nginx
```

### Удалить всё включая конфиги

```bash
sudo apt purge nginx
```

`purge` = вырвать с корнем.

### Удалить неиспользуемые зависимости

```bash
sudo apt autoremove
```

Удаляет пакеты которые были установлены как зависимости но больше не нужны.

> **Совет:** Делай `autoremove` после `remove` — освободишь место.

---

## 13.6 Очистка кэша

apt хранит скачанные пакеты в кэше:

```bash
ls /var/cache/apt/archives/
```

### Очистить кэш

```bash
sudo apt clean
```

Освободит место. Пакеты можно скачать снова если нужно.

## 13.7 Сторонние репозитории (PPA)

Не весь софт есть в стандартных репозиториях.

**PPA** (Personal Package Archive) — сторонний репозиторий.

### Добавить PPA

```bash
sudo add-apt-repository ppa:deadsnakes/ppa
sudo apt update
sudo apt install python3.12
```

### Удалить PPA

```bash
sudo add-apt-repository --remove ppa:deadsnakes/ppa
```

> **Опасно:** PPA — это сторонний источник. Доверяй только проверенным.

## 13.8 Snap пакеты

**Snap** — универсальный формат пакетов от Canonical.

Плюсы:
- Всегда свежая версия
- Работает на любом Linux
- Изолирован от системы

Минусы:
- Медленнее запускается
- Занимает больше места
- Не всё доступно

### Установить snap пакет

```bash
sudo snap install code --classic
```

### Посмотреть snap пакеты

```bash
snap list
```

> **Совет:** Для серверов используй `apt`. Snap — для десктопа.

## 13.9 Установка из исходников

Иногда пакета нет. Тогда собираешь сам.

```bash
# Скачать
wget https://example.com/software.tar.gz

# Распаковать
tar xzf software.tar.gz

# Перейти в директорию
cd software/

# Собрать и установить
./configure
make
sudo make install
```

> **Совет:** Избегай установки из исходников если есть пакет.
> Пакет легче обновлять и удалять.

## 13.10 Практика: установить стек для Python-приложения

Типичный набор для веб-приложения:

```bash
# Обновить систему
sudo apt update && sudo apt upgrade -y

# Веб-сервер
sudo apt install -y nginx

# База данных
sudo apt install -y postgresql postgresql-contrib

# Python и пакеты
sudo apt install -y python3 python3-pip python3-venv

# Утилиты
sudo apt install -y git curl htop

# Проверить всё работает
systemctl status nginx
systemctl status postgresql
python3 --version
```

---

## 13.11 Автоматические обновления

Можно настроить автообновления:

```bash
sudo apt install -y unattended-upgrades
sudo dpkg-reconfigure -plow unattended-upgrades
```

Будет обновлять безопасность автоматически.

> **Опасно:** Автообновления = удобно но рискованно.
> Большое обновление может сломать конфиги.
> Для продакшена — лучше обновлять вручную и проверять.

## 📝 Упражнения

### Упражнение 13.1: Обновление системы
**Задача:**
1. Обнови список пакетов: `sudo apt update`
2. Посмотри сколько программ можно обновить
3. Обнови всё: `sudo apt upgrade -y`
4. Удали ненужное: `sudo apt autoremove`

### Упражнение 13.2: Установка программ
**Задача:**
1. Установи htop: `sudo apt install -y htop`
2. Установи git: `sudo apt install -y git`
3. Проверь: `htop --version`, `git --version`

### Упражнение 13.3: Поиск и информация
**Задача:**
1. Найди все пакеты python3: `apt search python3 | head -20`
2. Посмотри информацию о nginx: `apt show nginx`
3. Посмотри что уже установлено: `apt list --installed | head -20`

### Упражнение 13.4: Установка и удаление
**Задача:**
1. Установи что-нибудь временное: `sudo apt install -y sl`
2. Запусти: `sl` (это паровозик 🚂)
3. Удали: `sudo apt remove sl`
4. Удали зависимости: `sudo apt autoremove`
