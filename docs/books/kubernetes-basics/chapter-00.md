# Глава 0: Зачем Kubernetes — честный разговор

> **Запомни:** K8s сложнее docker-compose. Но за каждым усложнением — реальная проблема которую оно решает.

---

## 0.1 Три проблемы docker-compose

### Проблема 1: Надёжность

```
Сервер упал → всё приложение недоступно
docker-compose не умеет "поднять на другом сервере"
```

### Проблема 2: Масштабирование

```
Трафик ×10 → нужно больше копий
docker-compose: scale возможен, но без балансировки
```

### Проблема 3: Обновления

```
docker-compose down → новый up → секунды даунтайма
Нет встроенного rolling update
```

---

## 0.2 Когда K8s НЕ нужен

- Один маленький сервис, один сервер, один разработчик
- docker-compose справляется
- Стартап на ранней стадии

---

## 0.3 Когда K8s нужен

- Несколько сервисов с разными требованиями
- Требования к надёжности (uptime 99.9%+)
- Нужно горизонтальное масштабирование
- Обновления без даунтайма

---

## 0.4 Архитектура кластера

```
Control Plane               Worker Nodes
┌─────────────────┐         ┌────────────┐
│ API Server      │──────→  │ Node 1     │
│ etcd (state)    │         │ Pods...    │
│ Scheduler       │──────→  │ Node 2     │
│ Controller Mgr  │         │ Pods...    │
└─────────────────┘         └────────────┘
```

| Компонент | Что делает |
|-----------|-----------|
| API Server | Принимает команды (kubectl) |
| etcd | Хранит состояние кластера |
| Scheduler | Решает где запустить Pod |
| Controller Manager | Следит за желаемым состоянием |

## 0.5 k3s — K8s для обучения

k3s =轻量级 Kubernetes. Один бинарник. Идеально для обучения.

### Установка

```bash
# На Ubuntu Server
curl -sfL https://get.k3s.io | sh -

# Проверить
kubectl get nodes
```

```
NAME     STATUS   ROLES    AGE   VERSION
myserver Ready    master   1m    v1.28.0+k3s1
```

> **Совет:** Для обучения k3s на VirtualBox достаточно.
> Для продакшена — managed K8s (облако) или k3s на VPS.

## 0.6 Docker vs Kubernetes

| Docker | Kubernetes |
|--------|-----------|
| `docker run` | Pod |
| `docker-compose.yml` | YAML-манифест |
| `docker restart` | Контроллер |
| `docker network` | Service |
| `docker volume` | PersistentVolume |
| `.env` | ConfigMap + Secret |

> **Паттерн обучения:** Открой второй терминал:
> ```bash
> watch kubectl get pods
> ```
> Все изменения видны в реальном времени. Используй ВСЕГДА.
