# Глава 8: DDoS, бот-сканирование и пределы локальной защиты

> **Запомни:** если атака забила твой канал до сервера, локальный firewall уже физически поздно срабатывает.

---

## 8.1 Разные типы внешнего мусора

### Бот-сканирование

Это постоянный фоновый шум интернета:
- поиск открытых портов;
- login brute-force;
- проверка стандартных путей и баннеров.

### Application-layer flood

Очень много HTTP-запросов к сайту или login endpoint.

### Volumetric DDoS

Слишком большой поток трафика, который забивает канал или edge раньше, чем сервер успевает что-либо осмыслить.

---

## 8.2 Что реально может Linux-сервер

Linux-сервер может:
- фильтровать соединения;
- rate-limit'ить часть HTTP-трафика через proxy;
- банить шумные IP;
- дать сигнал и логи.

Linux-сервер не может:
- спасти канал, который уже забит upstream-трафиком;
- в одиночку быть anti-DDoS-решением для серьёзной volumetric-атаки.

---

## 8.3 Где помогают разные слои

### Host firewall

Полезен против:
- лишних портов;
- простых соединений к ненужным сервисам.

### Reverse proxy + rate limit

Полезны против:
- части HTTP flood;
- brute-force на `/login`;
- дешёвого application-layer мусора.

### Gateway / cloud firewall

Полезны как ранний сетевой фильтр.

### CDN / WAF / provider anti-DDoS

Это уже внешний слой, который нужен, когда проблема превышает локальные ресурсы.

---

## 8.4 Практика: controlled load на своей lab

На своей lab можно аккуратно проверить, как ведёт себя сервис под небольшим всплеском запросов.

Например, на отдельной тестовой машине:

```bash
wrk -t2 -c20 -d10s http://HOST/
```

И параллельно смотреть:

```bash
top
ss -s
sudo tail -f /var/log/nginx/access.log
```

Важно: это не DDoS-учебник. Это controlled load test на своей инфраструктуре.

---

## 8.5 Что делать владельцу одного VPS

Реалистичный defensive baseline:

- не светить лишние сервисы;
- держать proxy на входе;
- иметь rate limiting на чувствительных endpoints;
- использовать CDN/WAF перед сайтом, если сервис реально публичный;
- иметь у провайдера понятный план действий при abuse/DDoS;
- знать, как быстро переключить сервис за внешнюю защиту.

---

## 8.6 Типовые ошибки

- надеяться, что `ufw` решит большой DDoS;
- путать controlled load test и разрушительную нагрузку;
- делать агрессивный rate limit и резать своих пользователей;
- игнорировать провайдера и CDN-слой;
- считать, что "у меня маленький сайт, значит меня не тронут".
