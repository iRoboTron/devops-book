## Для кого эта книга

- для домашней лаборатории;
- для владельца одного VPS;
- для small business, где один-два публичных сервера;
- для всех, кто слышал про firewall и WAF, но не понимает, как это собрать в систему.

---

## Предварительные требования

- базовый Linux;
- SSH и понимание ключей;
- базовые порты и IP-адреса;
- желательно опыт с `nginx` или `caddy`;
- готовность делать lab-only проверки только на своих системах.

---

## Главная идея

У публичного сервера нет одной волшебной настройки, которая делает его безопасным.

Есть слои:

```text
Интернет
  -> router / edge firewall
  -> reverse proxy / WAF / rate limiting
  -> Linux host firewall
  -> SSH hardening
  -> fail2ban / CrowdSec
  -> IDS/IPS / логи / сигналы
```

Каждый слой:
- сокращает поверхность атаки;
- мешает автоматическим ботам;
- уменьшает последствия ошибки;
- даёт сигнал защитнику.

## Что читатель получит к концу книги

- понятную threat model для публичного сервера;
- конфигурацию SSH только по ключам;
- firewall policy по умолчанию `deny incoming`;
- безопасную публикацию веб-сервиса;
- базовый reverse proxy + rate limiting;
- бан шумных попыток через fail2ban/CrowdSec;
- понимание, когда нужны IDS/IPS и когда они лишние;
- честную картину про DDoS: где заканчивается сила локального сервера;
- три варианта итогового проекта под разный масштаб.

## Как пользоваться книгой

1. Читай по порядку: темы завязаны друг на друга.
2. Каждую защиту проверяй только на своей lab.
3. После крупных шагов делай snapshot.
4. Не переходи к итоговому проекту, пока не умеешь читать логи SSH, firewall и proxy.

## Важная рамка безопасности

Все демонстрации в этой книге:
- только на своих VM, контейнерах, тестовых VPS и доменах;
- только как defensive validation;
- без вредоносного кода и без атак на чужие системы.
