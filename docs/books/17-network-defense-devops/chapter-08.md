# Глава 8: Практика по масштабам

> **Запомни:** сетевой дизайн нельзя оценивать в вакууме. Один и тот же принцип выглядит по-разному дома, на VPS и в small business.

---

## 8.1 Домашняя практика

Минимальный практический сценарий:
- роутер/гипервизор;
- 1-2 VM;
- отдельный серверный сегмент или хотя бы отдельная виртуальная сеть;
- controlled checks между VM.

Что проверить:
- какие сервисы доступны из клиентской VM;
- какие недоступны;
- как меняется картина после правил firewall/gateway.

---

## 8.2 Один VPS

Проверяй:
- какие порты реально видны извне;
- что закрыто cloud firewall/security group;
- что закрыто host firewall;
- как выглядит административный доступ: прямой SSH, VPN или bastion.

Цель — доказать, что твой VPS не "публичный по умолчанию".

---

## 8.3 Small business pattern

В lab это можно эмулировать так:

```
admin/client
  -> gateway/firewall
  -> server-net
  -> отдельный app host
```

Что проверять:
- админский доступ не смешан с пользовательским;
- IoT/пользовательские устройства не имеют прямого доступа к server zone;
- логика ACL понятна и воспроизводима.

---

## 8.4 Packet visibility

Сделай controlled captures:

```bash
sudo tcpdump -ni any host CLIENT_IP
```

Сравни:
- обычный `curl` к приложению;
- `nmap` со своей lab-машины;
- попытку доступа к запрещённому сервису.

Эта глава должна учить видеть network truth, а не только рисовать схемы.
