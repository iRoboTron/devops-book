# Приложение A: Шпаргалка и быстрые команды

> Краткий operational reference для книги: Сетевая защита и защитные устройства: Firewall, DMZ, VPN, IDS/IPS

---

## Что полезно держать под рукой

- схема активов и trust zones;
- список опубликованных портов;
- список административных входов;
- где лежат основные логи;
- какие controlled tests допустимы в этой книге.

---

## Базовая рабочая рутина защитника

1. Проверить текущую конфигурацию.
2. Снять baseline.
3. Внести изменение.
4. Проверить сервис.
5. Прогнать безопасный test.
6. Посмотреть логи и сигналы.
7. Зафиксировать результат.

---

## Полезные команды общего назначения

```bash
ss -tulpn
sudo journalctl -xe
sudo systemctl status SERVICE
sudo tail -f /var/log/auth.log
sudo tail -f /var/log/nginx/error.log
curl -I http://HOST
curl -vk https://HOST
```

## Напоминание

Эта шпаргалка не заменяет главы книги. Она помогает быстро вспомнить порядок проверки и не пропустить базовые defensive шаги.
