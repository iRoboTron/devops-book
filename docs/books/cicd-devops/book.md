## Что читатель настроит к концу книги

```yaml
Push в main → Тесты → Сборка Docker → Деплой на сервер
   │              │          │              │
   │              ✅         ✅             ✅
   ▼              ▼          ▼              ▼
 Разработчик   pytest   ghcr.io/image   docker pull
                                +       docker-compose up -d
```

Одна команда `git push` — и код автоматически проходит тесты, собирается в Docker-образ и деплоится на сервер. Без единого ручного шага.

## Главная идея

CI/CD — это **конвейер** который берёт твой код и доставляет его на сервер автоматически, каждый раз одинаково.

- **CI** (Continuous Integration) — автоматически проверяет что код не сломан
- **CD** (Continuous Delivery) — можно задеплоить одной кнопкой
- **CD** (Continuous Deployment) — деплой происходит автоматически
