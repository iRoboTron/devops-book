## 0.1 Чеклист production-сервера

Пройдись по пунктам. Честно.

### Конфигурация

- [ ] Все секреты в `.env` файле (не в docker-compose.yml)
- [ ] `.env` имеет права `600` (только владелец читает)
- [ ] `.env` НЕ в git (в `.gitignore`)
- [ ] `.env.example` в git (шаблон без значений)
- [ ] База данных НЕ доступна снаружи (нет `ports: "5432:5432"`)
- [ ] PostgreSQL работает от отдельного пользователя (не `postgres` суперпользователь)

### Бэкапы

- [ ] Бэкап базы данных настроен
- [ ] Бэкап загруженных файлов настроен
- [ ] Бэкап конфигов настроен
- [ ] Бэкапы хранятся НЕ только на сервере (есть копия вне площадки)
- [ ] Бэкап когда-нибудь проверялся на восстановление
- [ ] Старые бэкапы автоматически удаляются (ротация)

### Стабильность

- [ ] После `reboot` всё поднимается само (`restart: unless-stopped`)
- [ ] Healthcheck для базы данных настроен
- [ ] Логи ротируются (не растут бесконечно)
- [ ] Docker-мусор чистится автоматически
- [ ] Диск заполнен не более чем на 70%
- [ ] Swap настроен (на случай утечки памяти)

### Безопасность

- [ ] Nginx добавляет заголовки безопасности
- [ ] Rate limiting настроен
- [ ] Фаервол включён (ufw)
- [ ] HTTPS работает (не HTTP)
- [ ] SSH только по ключам (не по паролю)

## 0.2 Быстрая диагностика сервера

Зайди на сервер и выполни:

### Диск

```bash
df -h
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        50G   12G   36G  25% /
```

| Use% | Значение |
|------|----------|
| < 50% | Отлично |
| 50-70% | Норма |
| 70-85% | Пора чистить |
| > 85% | Критично |

### Память

```bash
free -h
              total   used   free   available
Mem:          3.9Gi  2.1Gi  1.2Gi  1.5Gi
Swap:         2.0Gi  0B     2.0Gi  2.0Gi
```

Если `available` < 500МБ — память под давлением.

### Контейнеры

```bash
docker ps
CONTAINER ID   IMAGE           STATUS          PORTS     NAMES
abc123         myapp:latest    Up 3 days       8000/tcp  myapp-app-1
def456         postgres:16     Up 3 days (healthy)        myapp-db-1
```

Все `Up`? Есть `(healthy)`?

### Потребление ресурсов

```bash
docker stats --no-stream
CONTAINER   CPU %    MEM USAGE / LIMIT
myapp-app   2.5%     120MiB / 512MiB
myapp-db    5.1%     340MiB / 1GiB
```

### Docker мусор

```bash
docker system df
TYPE            TOTAL     ACTIVE    SIZE      RECLAIMABLE
Images          15        3         2.1GB     1.8GB (85%)
Containers      8         2         0B        0B
Local Volumes   4         2         500MB     200MB (40%)
```

> **Запомни:** Reclaimable = можно удалить без вреда.
> Если > 1ГБ — время чистить.

## 0.3 Что будет если этого не сделать

| Не настроено | Что случится | Когда |
|-------------|-------------|-------|
| Бэкапы | Данные потеряны навсегда | При сбое диска |
| Восстановление не проверено | Бэкапы есть но не работают | Когда понадобишься |
| `postgres` суперпользователь | SQL-инъекция = полный доступ к БД | При атаке |
| Порт 5432 наружу | Любой подключится к базе | Сразу |
| Ротация логов | Диск заполнится → сервер упадёт | Через 3-6 месяцев |
| Docker мусор | Диск заполнится образами | Через 2-4 месяца |
| Restart policy | Сервисы не поднимутся после ребута | При перезагрузке |
| `.env` без защиты | Любой пользователь прочитает пароли | При компрометации |

> **Запомни:** Это не теория. Каждая строка — реальный инцидент.
> Книга поможет настроить всё до того как случится.
