## Что читатель настроит к концу книги

```
/opt/myapp/
├── docker-compose.yml
├── .env                    ← все секреты, chmod 600, в gitignore
├── nginx/
│   └── conf.d/app.conf     ← security headers, rate limiting
└── scripts/
    ├── backup.sh           ← бэкап БД + файлов, cron 3:00
    ├── restore.sh          ← восстановление из бэкапа
    └── health-check.sh     ← проверка что всё живо

/etc/cron.d/myapp-backup    ← ежедневный бэкап
/var/backups/myapp/         ← локальные бэкапы (7 дней)
  └── 2026-04-11/
      ├── db.sql.gz
      └── uploads.tar.gz
                   └→ rclone → Backblaze B2 (30 дней)
```

## Главная идея

**Сервер без бэкапа — это бомба с таймером.**

Три уровня надёжности:
1. Данные не теряются → бэкапы + проверенное восстановление
2. Сервис восстанавливается → правильные restart policy, healthcheck
3. Ты знаешь что происходит → управляемые конфиги, логи, переменные

## Как пользоваться книгой

1. **Читай по порядку** — главы строятся друг на друге
2. **Делай упражнения** — особенно восстановление из бэкапа (обязательно!)
3. **Отмечай чеклисты** — в конце каждой главы
4. **Практика восстановления** — закрой книгу и сделай на тестовой машине
