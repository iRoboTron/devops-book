# Глава 2: Первая команда и файловая система

> **Запомни:** В этой главе ты перестанешь бояться терминала. Команды — это просто слова. Ты говоришь Linux что сделать — он делает.

---

## 2.1 Где я? Команда `pwd`

Ты вошёл в систему. Перед тобой чёрный экран и мигающий курсор.

**Первый вопрос:** в какой директории я нахожусь?

```bash
pwd
```

`pwd` = **P**rint **W**orking **D**irectory (покажи рабочую директорию)

**Пример:**
```bash
adelfos@ubuntu-server:~$ pwd
/home/adelfos
```

**Что произошло:**
- Ты написал `pwd`
- Нажал Enter
- Linux показал полный путь до текущей директории
- Снова показал приглашение

> **Запомни:** Каждая команда работает так:
> 1. Ты пишешь команду
> 2. Нажимаешь Enter
> 3. Linux выполняет
> 4. Linux показывает результат (или молчит если всё ок)
> 5. Снова показывает приглашение

## 2.2 Что здесь есть? Команда `ls`

Ты в домашней директории. Что внутри?

```bash
ls
```

`ls` = **l**i**s**t (показать список)

**Пример:**
```bash
adelfos@ubuntu-server:~$ ls
Документы  Загрузки  Изображения
```

Пусто? Это нормально — свежая установка.

### Опции команды `ls`

Команды можно уточнять **опциями**. Опции начинаются с `-`.

```bash
ls -l
```

`-l` = длинный формат (подробно)

**Пример вывода:**
```
total 0
drwxr-xr-x 2 adelfos adelfos 4096 апр  9 10:30 Документы
drwxr-xr-x 2 adelfos adelfos 4096 апр  9 10:30 Загрузки
```

Сейчас не пугайся — права разберём в Главе 4. Важно что `-l` даёт **больше информации**.

```bash
ls -a
```

`-a` = все файлы (включая скрытые)

**Пример:**
```bash
adelfos@ubuntu-server:~$ ls -a
.  ..  .bashrc  .profile  Документы  Загрузки
```

Файлы начинающиеся с `.` — **скрытые**. Это конфигурационные файлы.

```bash
ls -la
```

Можно **комбинировать** опции: `-l` + `-a` = `-la`

Покажи всё и подробно. Самая частая форма `ls`.

> **Совет:** Не помнишь что делает опция? Посмотри помощь:
> ```bash
> ls --help
> ```
> Или человек-страницу:
> ```bash
> man ls
> ```
> (нажми `q` чтобы выйти из `man`)

## 2.3 Как перемещаться? Команда `cd`

`cd` = **c**hange **d**irectory (сменить директорию)

### Перейти в папку

```bash
cd Документы
```

Теперь ты в `/home/adelfos/Документы`. Проверь:

```bash
pwd
/home/adelfos/Документы
```

### Вернуться назад

```bash
cd ..
```

`..` = родительская директория (на уровень выше)

### Вернуться домой

```bash
cd
```

Или:

```bash
cd ~
```

`~` = домашняя директория текущего пользователя

### Перейти по абсолютному пути

```bash
cd /etc
```

Абсолютный путь начинается с `/` — отсчёт от корня системы.

### Перейти по относительному пути

```bash
cd Документы/работа
```

Относительный путь — от текущей директории.

### Специальные обозначения

| Обозначение | Значение |
|-------------|----------|
| `.` | Текущая директория |
| `..` | Родительская директория |
| `~` | Домашняя директория |
| `/` | Корень системы |

**Примеры:**

```bash
cd ~           # домой
cd ..          # на уровень выше
cd ../..       # на два уровня выше
cd -           # вернуться туда где был только что
cd /           # в корень системы
```

> **Совет:** `cd -` — недооценённая команда. Переключает между текущей и предыдущей директорией. Как кнопка "назад" в браузере.

## 2.4 Абсолютные и относительные пути

Это **критически важная** концепция.

### Абсолютный путь

Начинается от корня `/`. Всегда работает откуда угодно.

```bash
/home/adelfos/Документы/отчёт.txt
/etc/nginx/nginx.conf
/var/log/syslog
```

**Плюсы:**
- Точный — всегда приведёт куда нужно
- Работает из любой директории

**Минусы:**
- Длинный
- Неудобный если часто используешь

### Относительный путь

От текущей директории. Не начинается с `/`.

```bash
# Ты в /home/adelfos
Документы/отчёт.txt

# Ты в /home/adelfos/Документы
отчёт.txt

# Ты в /home/adelfos/Документы
../Загрузки/файл.zip
```

**Плюсы:**
- Короткий
- Удобный, когда работаешь в одной области

**Минусы:**
- Зависит от того где ты сейчас
- Можно ошибиться

> **Запомни:** Абсолютный путь не зависит от того, где ты сейчас. Относительный — зависит.
> - Начинается с `/` — абсолютный (от корня)
> - Не начинается с `/` — относительный (отсюда)
> - Начинается с `~` — домашняя директория (тоже абсолютный)

## 2.5 Файловая система Linux: карта территории

В Windows ты привык к дискам: `C:\`, `D:\`.
В Linux — **один корень** `/`. Всё остальное — внутри него.

```
/                          ← корень системы
├── bin/                   ← основные команды (ls, cp, mkdir)
├── boot/                  ← файлы загрузчика
├── dev/                   ← устройства (диски, USB)
├── etc/                   ← конфигурационные файлы
├── home/                  ← домашние папки пользователей
│   └── adelfos/           ← твоя домашняя папка
│       ├── Документы/
│       ├── Загрузки/
│       └── .bashrc        ← скрытый конфиг
├── lib/                   ← библиотеки (как DLL в Windows)
├── media/                 ← автомонтирование (флешки)
├── mnt/                   ← ручное монтирование
├── opt/                   ← дополнительный софт
├── proc/                  ← информация о процессах
├── root/                  ← домашняя папка root
├── run/                   ← временные файлы процессов
├── sbin/                  ← системные команды (для root)
├── tmp/                   ← временные файлы (очищаются!)
├── usr/                   ← программы и библиотеки пользователей
│   ├── bin/               ← пользовательские команды
│   ├── lib/               ← библиотеки
│   └── share/             ← общие данные
└── var/                   ← переменные данные (логи, кэш)
    └── log/               ← лог-файлы (здесь мы будем часто бывать)
```

### Какие директории тебе важны как DevOps'у

| Директория | Зачем |
|------------|-------|
| `/etc/` | Конфиги всех сервисов |
| `/var/log/` | Логи — сюда смотришь когда что-то сломалось |
| `/home/` | Домашние папки пользователей |
| `/tmp/` | Временные файлы |
| `/opt/` | Сторонний софт |
| `~` (твоя домашняя) | Твои файлы, скрипты, ключи |

> **Совет:** Не сохраняй рабочие файлы в системные директории. Домашняя папка — твоё безопасное место.

## 2.6 Автодополнение: твой лучший друг

Не печатай полные имена файлов. Используй **Tab**.

```bash
cd Док<Tab>
```

Linux допишет: `cd Документы/`

Если вариантов несколько — нажми Tab дважды:

```bash
cd D<Tab><Tab>
```

Покажет все варианты начинающиеся на D.

> **Запомни:** Tab экономит время и спасает от опечаток. Пользуйся всегда.

## 2.7 История команд

Ты не обязан переписывать команды.

**Стрелка вверх** ↑ — предыдущая команда
**Стрелка вниз** ↓ — следующая команда

```bash
history     # показать всю историю команд
```

**Поиск в истории:**
Нажми `Ctrl+R` — начни печатать — Linux найдёт команду которую ты раньше вводил.

```bash
(reverse-i-search)`pwd': pwd
```

Нашёл? Нажми Enter — выполни. Или стрелку — отредактируй.

## 2.8 Очистка экрана

Экран забился выводом? Очисти:

```bash
clear
```

Или нажми `Ctrl+L`.
