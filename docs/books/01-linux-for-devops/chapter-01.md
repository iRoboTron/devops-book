# Глава 1: Что такое Linux и зачем он DevOps'у

> **Запомни:** Эта глава — фундамент. Здесь не нужно запоминать команды. Здесь нужно понять **зачем** мы вообще лезем в терминал, когда есть красивые кнопки и мышки.

---

## 1.1 Краткая история Linux (и почему это важно)

В 1991 году финский студент Линус Торвальдс написал ядро операционной системы. Он выложил его в открытый доступ с простой идеей: **пусть каждый может использовать, смотреть код и улучшать**.

Эта идея изменила мир.

Сегодня Linux — это:
- Основная операционная система для серверов и облачной инфраструктуры
- Стандартная среда для VPS, Docker, Kubernetes и большинства production-систем
- Основа Android и множества встраиваемых устройств
- Сердце облачных технологий (AWS, Google Cloud, Azure)

> **Почему это важно:** Когда ты пишешь код на Python и хочешь чтобы он работал на сервере — в подавляющем большинстве случаев этот сервер будет на Linux. Ты не можешь эффективно разрабатывать для того, чего не понимаешь.

### Что такое ядро?

**Ядро (kernel)** — это сердце операционной системы. Оно:
- Управляет процессором (какая программа сколько ресурсов получает)
- Работает с памятью (кто сколько ОЗУ может взять)
- Общается с железом (диски, сеть, USB)
- Контролирует безопасность (кто что может делать)

Linux — это **ядро**. Не полная операционная система, а именно ядро.

Полная система называется **дистрибутив (distro)** — это ядро + утилиты + программы + настройки.

## 1.2 Дистрибутивы: какой выбрать и почему

Дистрибутивов Linux — сотни. Вот основные семейства:

### Семейство Debian/Ubuntu
- **Ubuntu** — самый популярный для серверов
- **Debian** — стабильный, консервативный
- **Linux Mint** — для десктопа

Пакетный менеджер: `apt` (Advanced Package Tool)

### Семейство Red Hat
- **RHEL (Red Hat Enterprise Linux)** — корпоративный, платный
- **CentOS/Rocky Linux** — бесплатный аналог RHEL
- **Fedora** — для энтузиастов

Пакетный менеджер: `yum` / `dnf`

### Семейство Arch
- **Arch Linux** — для тех кто хочет всё контролировать
- **Manjaro** — более дружелюбный Arch

Пакетный менеджер: `pacman`

### Другие
- **Alpine Linux** — минимальный (5МБ!), популярен в Docker
- **openSUSE** — европейский серверный дистрибутив

---

> **Совет:** Для обучения и серверов выбирай **Ubuntu Server**. Причины:
> - Огромное сообщество (любой гуглится)
> - Долгая поддержка (LTS версии — 5 лет обновлений)
> - Стандарт для веба
> - Все примеры в этой книге — для Ubuntu

## 1.3 Что такое терминал и почему он лучше GUI для DevOps

### GUI (Graphical User Interface)

Это то к чему ты привык: окна, кнопки, мышка, меню.
- Открыл файловый менеджер → кликнул на папку → нашёл файл
- Открыл браузер → ввёл адрес → нажал Enter

Удобно. Наглядно. Но **не масштабируется**.

### CLI (Command Line Interface)

Это терминал. Ты пишешь команды текстом.
- Хочешь создать 100 папок? Одна строка, 2 секунды.
- Хочешь найти все файлы больше 100МБ? Одна команда.
- Хочешь обновить 50 серверов одновременно? Один скрипт.

### Почему DevOps'ы любят терминал

**1. Автоматизация**
Всё, что ты пишешь в терминале, можно записать в скрипт.
Скрипт = рецепт. Рецепт = можно повторить 1000 раз одинаково.

**2. Удалённая работа**
Сервер может быть в дата-центре за 3000 км.
Ты не можешь подключить к нему монитор.
Но ты можешь подключиться по SSH и работать в терминале.

**3. Скорость**
Когда знаешь команды — работаешь в 10 раз быстрее.
Мышка медленнее клавиатуры. Это доказано.

**4. Надёжность**
На сервере может не быть графического интерфейса.
Там только терминал. Всегда.

> **Запомни:** GUI — для удобства одного пользователя. CLI — для автоматизации, скорости и серверов. DevOps выбирает CLI.

## 1.4 Как устроен сервер: простое объяснение

Сервер — это обычный компьютер. Просто без монитора, без мышки и без клавиатуры. Но с:
- **Процессором** (CPU) — думает
- **Памятью** (RAM) — хранит то с чем работает прямо сейчас
- **Диском** (HDD/SSD) — хранит всё навсегда
- **Сетью** (Network) — общается с миром

На этом компьютере стоит операционная система (обычно Linux).

На Linux работают **сервисы** (их ещё называют **демонами**):

| Сервис | Что делает |
|--------|-----------|
| Nginx/Apache | Отдаёт веб-страницы |
| PostgreSQL/MySQL | Хранит данные в базе |
| SSH | Позволяет подключаться удалённо |
| Docker | Запускает контейнеры |
| Nextcloud | Хранит файлы (как Google Drive) |

> **Пример:** Твой сервер с Nextcloud — это:
> - Железо (процессор, память, диск)
> - Linux (Ubuntu)
> - Сервисы: Nextcloud, возможно Docker, Nginx, база данных

Твоя задача как DevOps'а — заставить всё это работать вместе, надёжно и автоматически.

## 1.5 Установка Ubuntu Server: пошаговая инструкция

> **Важно:** Если ты устанавливаешь на виртуальную машину — это идеальный вариант для обучения. Ничего не сломаешь.

### Что нужно

- **VirtualBox** (или другой гипервизор): https://www.virtualbox.org/
- **Ubuntu Server ISO**: https://ubuntu.com/download/server
- Выбирай **LTS версию** (например, 24.04 LTS) — LTS = Long Term Support

### Шаг 1: Создаём виртуальную машину

1. Открой VirtualBox → нажми **Создать**
2. Имя: `ubuntu-server-devops`
3. Тип: **Linux**
4. Версия: **Ubuntu (64-bit)**
5. Память (RAM): **2048 МБ** (2 ГБ — минимум для сервера)
6. Жёсткий диск: **Создать новый** → **VDI** → **Динамический** → **25 ГБ**

> **Совет:** 25 ГБ хватит для обучения. Реальные серверы обычно от 50 ГБ.

### Шаг 2: Подключаем ISO-образ

1. Выбери созданную машину → **Настроить**
2. **Носители** → выбери пустой диск (значок CD)
3. Справа нажми на иконку диска → **Выбрать файл диска**
4. Найди скачанный `.iso` файл Ubuntu Server

### Шаг 3: Запускаем установку

1. Нажми **Запустить**
2. Появится экран установки — выбери язык **English** (серверы обычно на английском)

### Шаг 4: Проходим установщик

| Экран | Что выбрать | Почему |
|-------|------------|--------|
| Язык | English | Стандарт для серверов |
| Раскладка клавиатуры | English (US) | Меньше проблем с символами |
| Сеть | По умолчанию (DHCP) | IP получит автоматически |
| Прокси | Оставить пустым | Не нужно для домашней сети |
| Зеркало архива | По умолчанию | Ubuntu сама выберёт быстрое |
| Разметка диска | **Use entire disk** | Вся виртуалка — для Ubuntu |
| Профиль пользователя | Введи своё имя и пароль | Запомни пароль! |
| SSH Server | ✅ **Установить OpenSSH** | Обязательно для DevOps! |
| Snap-пакеты | Пропустить | Не нужно сейчас |

> **Запомни пароль!** Без него ты не войдёшь в систему. Запиши его, если нужно.

### Шаг 5: Первая загрузка

После установки система перезагрузится.

Ты увидишь чёрный экран с белым текстом:

```
Ubuntu 24.04 LTS ubuntu-server-devops tty1

ubuntu-server-devops login: _
```

Это — **консоль сервера**. Без графики. Только текст.

Введи свой логин и пароль.

После входа ты увидишь что-то вроде:

```
Welcome to Ubuntu 24.04 LTS (GNU/Linux 6.8.0-31-generic x86_64)

 * Documentation:  https://help.ubuntu.com
 * Management:     https://landscape.canonical.com
 * Support:        https://ubuntu.com/pro

0 updates can be applied immediately.

Last login: Thu Apr  9 10:30:00 UTC 2026
adelfos@ubuntu-server-devops:~$
```

**Поздравляю!** Ты внутри Linux-сервера.

## 1.6 Разбираемся что мы видим

Посмотри на последнюю строку:

```
adelfos@ubuntu-server-devops:~$
```

Это **приглашение командной строки (prompt)**. Оно говорит тебе:

| Часть | Значение |
|-------|----------|
| `adelfos` | Имя текущего пользователя |
| `@` | разделитель |
| `ubuntu-server-devops` | Имя сервера (hostname) |
| `:` | разделитель |
| `~` | Текущая директория (`~` = домашняя папка) |
| `$` | Ты — обычный пользователь (не root) |

> **Запомни:** Смотри на символ в конце приглашения и на текущую директорию.
> - `$` в конце = обычный пользователь
> - `#` в конце = root (администратор)
> - `~` = твоя домашняя папка (`/home/adelfos`)

## 1.7 Кто такой root и почему не надо под ним работать

**root** — это суперпользователь. Бог Linux-системы.
- Может читать любые файлы
- Может удалять что угодно (включая системные файлы)
- Может устанавливать программы
- Может ломать систему одной командой

```bash
# Эта команда удалит ВСЁ на сервере без вопросов:
rm -rf /
```

> **Опасно:** Никогда не работай под root без крайней необходимости.
> Используй своего пользователя + `sudo` когда нужно.

### Правило DevOps'а:

> Если команда может что-то сломать — используй `sudo`.
> Если команда безопасна — работай без `sudo`.

Это защищает тебя от случайных ошибок.

## 1.8 Терминал: базовые понятия

### Что такое оболочка (shell)?

**Shell** — это программа, которая:
1. Показывает тебе приглашение (`adelfos@server:~$`)
2. Читает, что ты написал
3. Выполняет команду
4. Показывает результат
5. Снова показывает приглашение

Самая распространённая оболочка — **bash** (Bourne Again Shell).
В Ubuntu она стоит по умолчанию.

### Как писать команды

```bash
имя_команды [аргументы] [опции]
```

- **имя_команды** — что сделать (`ls`, `cd`, `mkdir`)
- **аргументы** — над чем сделать (`/home`, `file.txt`)
- **опции** — как сделать (`-l`, `-a`, `--help`)

### Пример

```bash
ls -la /home
```

| Часть | Значение |
|-------|----------|
| `ls` | Команда: "покажи содержимое" |
| `-la` | Опции: "подробно (`-l`) и скрытые файлы (`-a`)" |
| `/home` | Аргумент: "директория /home" |

> **Совет:** Не помнишь опции? Добавь `--help` к любой команде:
> ```bash
> ls --help
> ```

## 1.9 Переход к практике

Теория без практики — мертва.

В следующих главах мы начнём работать с командной строкой по-настоящему. А пока — вот что нужно понять:

**Ты теперь администратор сервера.**

Не пользователя который кликает на иконки.
Не разработчика который пишет код в IDE.
А **администратора** который:
- Настраивает систему
- Управляет сервисами
- Автоматизирует рутину
- Диагностирует проблемы

Это другая роль. Другие навыки. Другой способ мышления.

И мы начинаем его развивать. Прямо сейчас.

### Упражнение 1.2: Вход в систему
**Задача:** Войди в систему под своим пользователем.

**Что проверить:**
- Видишь приветствие Ubuntu?
- Видишь приглашение командной строки?
- В конце `$` (не `#`)?

### Упражнение 1.3: Проверка SSH
**Задача:** Убедись что SSH установлен.

Выполни команду:
```bash
sudo systemctl status ssh
```

Ты увидишь много текста. Ищи зелёный кружок или слово **active**.

> **Не пугайся:** `sudo` попросит пароль. Это нормально — вводи пароль пользователя.
> Когда вводишь пароль в терминале, **символы не отображаются**. Это нормально и сделано для безопасности.

### Упражнение 1.4: Кто я?
**Задача:** Узнай своё имя пользователя и имя сервера.

Выполни:
```bash
whoami
hostname
```

Запомни эти два значения. Они будут нужны постоянно.
