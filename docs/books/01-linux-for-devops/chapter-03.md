# Глава 3: Файлы и директории

> **Запомни:** В Linux **всё** — файл. Даже устройства, процессы и настройки. Научись управлять файлами — управляешь системой.

---

## 3.1 Создать директорию: `mkdir`

`mkdir` = **m**a**k**e **dir**ectory (создать директорию)

```bash
mkdir projects
```

Создал папку `projects` в текущей директории.

Проверь:

```bash
ls
projects
```

### Создать несколько директорий

```bash
mkdir project1 project2 project3
```

### Создать вложенную директорию (ключ `-p`)

```bash
mkdir -p projects/myapp/src
```

`-p` = создать все промежуточные директории если их нет.

Без `-p` это бы упало с ошибкой, если `projects` или `myapp` не существуют.

> **Совет:** Используй `-p` всегда когда создаёшь вложенные папки. Это безопаснее и быстрее.

### Что может пойти не так

**Ошибка: `File exists`**
```bash
mkdir projects
mkdir: невозможно создать каталог «projects»: Файл существует
```
Папка уже есть. Выбери другое имя или удали существующую.

**Ошибка: `Permission denied`**
```bash
mkdir /etc/myconfig
mkdir: невозможно создать каталог «/etc/myconfig»: Отказано в доступе
```
Нет прав. Нужен `sudo` если пишешь в системную директорию. Но лучше не писать туда.

## 3.2 Создать файл: `touch`

```bash
touch README.md
```

`touch` делает две вещи:
1. Если файл **не существует** — создаёт пустой
2. Если файл **существует** — обновляет время изменения

**Пример:**
```bash
adelfos@ubuntu-server:~$ ls
projects
adelfos@ubuntu-server:~$ touch README.md
adelfos@ubuntu-server:~$ ls
projects  README.md
```

> **Совет:** `touch` удобен когда нужно быстро создать файл для тестов. Для реальных файлов ты обычно используешь редактор (nano, vim).

### Создать несколько файлов

```bash
touch file1.txt file2.txt file3.txt
```

## 3.3 Скопировать: `cp`

`cp` = **c**o**p**y (копировать)

### Копировать файл

```bash
cp README.md README-backup.md
```

Формат: `cp откуда куда`

### Копировать в другую директорию

```bash
cp README.md projects/
```

Файл скопируется в `projects/README.md`.

### Копировать директорию (ключ `-r`)

```bash
cp -r projects projects-backup
```

`-r` = рекурсивно (всё содержимое включая подпапки)

**Без `-r` копирование директории не работает.**

### Копировать с сохранением прав и времени

```bash
cp -a projects projects-backup
```

`-a` = archive (сохранить всё: права, владельцев, время)

> **Запомни:** У `cp` всегда два главных вопроса: что копируем и куда копируем.
> - Файлы: `cp source dest`
> - Директории: `cp -r source dest`
> - Точная копия: `cp -a source dest`

### Что может пойти не так

**Ошибка: `omitting directory`**
```bash
cp projects projects-backup
cp: -r not specified; omitting directory 'projects'
```
Забыл `-r`. Это частая ошибка — запомни её.

## 3.4 Переместить и переименовать: `mv`

`mv` = **m**o**v**e (переместить)

### Переименовать

```bash
mv README.md README-old.md
```

В Linux переименование = перемещение с новым именем. Одна команда.

### Переместить файл

```bash
mv README.md projects/
```

Файл переместился в директорию `projects/`.

### Переместить директорию

```bash
mv projects /opt/
```

Не нужен `-r` — `mv` работает с директориями без дополнительных ключей.

### Что может пойти не так

**Файл перезаписан без предупреждения:**
```bash
mv important.txt existing-file.txt
```
Если `existing-file.txt` существует — он будет **перезаписан**. Без вопросов.

> **Совет:** Используй `-i` (interactive) чтобы получить предупреждение:
> ```bash
> mv -i important.txt existing-file.txt
> mv: перезаписать «existing-file.txt»?
> ```

## 3.5 Удалить: `rm`

`rm` = **r**e**m**ove (удалить)

### Удалить файл

```bash
rm README.md
```

### Удалить несколько файлов

```bash
rm file1.txt file2.txt file3.txt
```

### Удалить директорию и всё содержимое (ключ `-r`)

```bash
rm -r projects/
```

> **Опасно:** `rm -r` удаляет **всё** в директории без возможности восстановления. Нет корзины. Нет Ctrl+Z.

### Удалить с подтверждением (ключ `-i`)

```bash
rm -i important.txt
rm: удалить обычный файл «important.txt»?
```

`-i` = спросить перед каждым удалением.

### Удалить пустую директорию

```bash
rmdir empty_folder
```

`rmdir` работает только с **пустыми** директориями. Безопаснее чем `rm -r`.

### ОПАСНАЯ КОМАНДА: `rm -rf /`

```bash
rm -rf /
```

- `-r` = рекурсивно
- `-f` = force (без вопросов)
- `/` = корень системы

**Эта команда удалит ВЕСЬ СЕРВЕР без возможности восстановления.**

> **Опасно:** Никогда не запускай `rm -rf` не проверив путь дважды.
> Опечатка в одном символе — и ты потеряешь всё.
> 
> Было много случаев когда разработчики случайно удаляли продакшен серверы.

## 3.6 Прочитать файл: `cat`

`cat` = **cat**enate (показать содержимое)

```bash
cat README.md
Hello, this is my project!
```

Показывает **весь** файл сразу. Если файл большой — мелькнёт и всё.

### Объединить несколько файлов

```bash
cat file1.txt file2.txt
```

Покажет содержимое обоих файлов подряд.

### Записать вывод в другой файл

```bash
cat file1.txt file2.txt > combined.txt
```

`>` = перенаправить вывод в файл.

> **Запомни:** `>` создаёт новый файл (или перезаписывает существующий).

---

## 3.7 Читать постранично: `less`

```bash
less /var/log/syslog
```

`less` показывает файл **поэкранно**.

**Управление:**
| Клавиша | Действие |
|---------|----------|
| `Пробел` | Следующая страница |
| `b` | Предыдущая страница |
| `↓` / `↑` | Построчно |
| `/слово` | Поиск слова |
| `n` | Следующий результат поиска |
| `q` | Выйти |

> **Совет:** Используй `less` для больших файлов. `cat` — только для маленьких.

## 3.8 Начало и конец файла: `head` и `tail`

### Первые 10 строк

```bash
head /var/log/syslog
```

### Первые N строк

```bash
head -n 20 /var/log/syslog
```

Покажет первые 20 строк.

### Последние 10 строк

```bash
tail /var/log/syslog
```

### Последние N строк

```bash
tail -n 20 /var/log/syslog
```

### Следить за файлом в реальном времени

```bash
tail -f /var/log/syslog
```

`-f` = follow (следить). Экран обновляется когда в файл добавляются новые строки.

> **Запомни:** `tail -f` — твой лучший друг когда смотришь логи работающего сервиса.
> `Ctrl+C` чтобы остановить.

## 3.9 Найти файлы: `find`

`find` ищет файлы по имени, типу, размеру, дате.

### Найти по имени

```bash
find /home -name "README.md"
```

Ищет файл `README.md` в `/home` и всех поддиректориях.

### Найти все `.txt` файлы

```bash
find /home -name "*.txt"
```

`*` = любой набор символов (wildcard)

### Найти директории

```bash
find /home -type d
```

`-type d` = только директории (directory)

### Найти только файлы

```bash
find /home -type f
```

`-type f` = только файлы (file)

### Найти файлы больше 100МБ

```bash
find / -size +100M
```

Полезно когда диск забит — ищешь что занимает место.

### Найти и выполнить действие

```bash
find /home -name "*.log" -delete
```

Нашёл все `.log` файлы и удалил.

> **Опасно:** `-delete` удаляет без подтверждения. Сначала проверь что найдётся без `-delete`.

## 3.10 Wildcards: шаблоны имён

В Linux есть специальные символы для шаблонов:

| Символ | Значение | Пример |
|--------|----------|--------|
| `*` | Любые символы (0 или больше) | `*.txt` = все txt файлы |
| `?` | Один любой символ | `file?.txt` = file1.txt, fileA.txt |
| `[abc]` | Один из символов | `file[123].txt` |
| `[a-z]` | Диапазон символов | `file[a-z].txt` |

**Примеры:**

```bash
ls *.md          # все markdown файлы
ls *.log         # все лог файлы
ls config*       # всё что начинается на config
rm *.tmp         # удалить все временные файлы
```

> **Совет:** Перед `rm` с wildcard сначала проверь что найдётся с `ls`:
> ```bash
> ls *.tmp        # сначала посмотри
> rm *.tmp        # потом удаляй
> ```

## 3.11 Скрытые файлы

Файлы начинающиеся с `.` — скрытые. `ls` их не показывает без `-a`.

```bash
ls -a
.   ..  .bashrc  .profile  .ssh
```

Зачем? Чтобы не мешались. Это конфигурационные файлы.

### Важные скрытые файлы

| Файл | Зачем |
|------|-------|
| `~/.bashrc` | Настройки терминала для пользователя |
| `~/.profile` | Настройки окружения |
| `~/.ssh/` | SSH-ключи |
| `~/.config/` | Конфигурации программ |
| `~/.bash_history` | История команд |

> **Запомни:** Не удаляй скрытые файлы если не знаешь что они делают. Особенно `.bashrc` и `.ssh`.

## 3.12 Перенаправление и конвейеры

До сих пор вывод команд шёл на экран. Это не всегда удобно.

### Три потока

Каждая команда имеет три канала:

| Поток | Название | Куда по умолчанию |
|-------|----------|-------------------|
| `stdin` (0) | Ввод | Клавиатура |
| `stdout` (1) | Вывод | Экран |
| `stderr` (2) | Ошибки | Экран |

Ты можешь перенаправить любой из них в файл или другую команду.

### `>` — записать в файл (перезаписать)

```bash
echo "Hello" > greeting.txt
cat greeting.txt
Hello
```

`>` берёт вывод команды и пишет в файл. Если файл существует — **перезаписывает**.

```bash
echo "First line" > file.txt
echo "Second line" > file.txt
cat file.txt
Second line
```

Первая строка потерялась. `>` перезаписал файл.

### `>>` — дописать в конец

```bash
echo "First line" > file.txt
echo "Second line" >> file.txt
cat file.txt
First line
Second line
```

`>>` добавляет в конец, не трогая существующее содержимое.

> **Совет:** `>` = «с нуля». `>>` = «дополнить».
> Перепутаешь — потеряешь данные.

### `2>` — ошибки в файл

```bash
find / -name "secret" 2>/dev/null
```

`2>` = перенаправить stderr (поток ошибок).

`/dev/null` — «чёрная дыра». Всё, что туда попадает, исчезает навсегда.

Это удобно когда errors не важны.

### `2>&1` — ошибки туда же куда вывод

```bash
command > output.log 2>&1
```

`2>&1` = направь stderr туда же куда stdout.

Теперь и обычный вывод и ошибки — в одном файле.

> **Совет:** Для логов используй:
> ```bash
> ./deploy.sh > deploy.log 2>&1
> ```
> Всё в одном файле — легче диагностировать.

### `|` — конвейер (pipe)

**Pipe** — передача вывода одной команды на вход другой.

```bash
команда1 | команда2 | команда3
```

Вывод `команда1` становится вводом для `команда2`.

### Пример: найти процесс

```bash
ps aux | grep nginx
```

`ps aux` выдал все процессы → `grep nginx` отфильтровал только nginx.

Без pipe пришлось бы искать глазами среди сотен строк.

### Пример: последние ошибки

```bash
cat /var/log/syslog | grep -i "error" | tail -20
```

1. `cat` — прочитал весь лог
2. `grep -i "error"` — отфильтровал строки с "error"
3. `tail -20` — показал последние 20

Цепочка из трёх команд — и ты видишь только нужное.

### Пример: подсчёт

```bash
ls -la /etc | wc -l
156
```

`wc -l` = **w**ord **c**ount **-l**ines (посчитать строки).

Сколько файлов в `/etc`? 156.

### Пример: сортировка

```bash
du -sh /var/* | sort -rh | head -5
1.2G    /var/lib
456M    /var/log
234M    /var/cache
```

1. `du` — размеры директорий
2. `sort -rh` — отсортировать по числам, обратно
3. `head -5` — топ 5

Самые большие директории за одну команду.

### Цепочки команд — сила DevOps'а

Типичные цепочки:

```bash
# Сколько процессов nginx?
ps aux | grep nginx | wc -l

# Найти большие логи
find /var/log -name "*.log" -size +10M | sort

# Кто занимает больше всего памяти
ps aux --sort=-%mem | head -10

# Посчитать уникальные IP в логе Nginx
awk '{print $1}' /var/log/nginx/access.log | sort -u | wc -l
```

> **Запомни:** Pipe — главный инструмент DevOps'а.
> Одна команда делает одну вещь. Pipe соединяет их в мощный инструмент.

### Визуально

```
┌──────────┐    stdout    ┌──────────┐    stdout    ┌──────────┐
│   ps aux   │ ──────────► │ grep nginx│ ──────────► │  wc -l   │
└──────────┘              └──────────┘              └──────────┘
                                                      │
                                                      ▼
                                                    "3"
```

Каждая команда — конвейер. Вывод одной — ввод для следующей.

## 📝 Упражнения

### Упражнение 3.1: Создание структуры проекта
**Задача:** Создай структуру проекта из командной строки:

```
~/myproject/
├── src/
│   ├── main.py
│   └── utils.py
├── tests/
│   └── test_main.py
├── docs/
│   └── README.md
└── config/
    └── settings.ini
```

**Подсказка:** Используй `mkdir -p` и `touch`.

### Упражнение 3.2: Копирование и перемещение
**Задача:**
1. Создай директорию `~/backup`
2. Скопируй `~/myproject` в `~/backup/myproject`
3. Убедись что скопировалось (`ls -la ~/backup/myproject`)
4. Перемести `~/myproject/config` в `~/myproject/configs` (переименуй)

### Упражнение 3.3: Чтение файлов
**Задача:**
1. Создай файл с текстом: `echo "Hello World" > ~/hello.txt`
2. Прочитай его через `cat`
3. Посмотри последние строки системного лога: `tail /var/log/syslog`
4. (Если лог пустой — создай свой файл и добавь строки)

### Упражнение 3.4: Поиск файлов
**Задача:**
1. Найди все `.py` файлы в своей домашней директории
2. Найди все директории в `/var`
3. Найди все файлы больше 50МБ на всём сервере (подсказка: нужен `sudo`)

### Упражнение 3.5: Работа с wildcard
**Задача:**
1. Создай 5 файлов: `touch file{1,2,3,4,5}.txt`
2. Посмотри их: `ls *.txt`
3. Удали файлы 2 и 4: `rm file{2,4}.txt`
4. Убедись что остались только 1, 3, 5

### Упражнение 3.6: Перенаправление и pipe
**Задача:**
1. Создай файл: `echo "line 1" > test.txt`
2. Допиши: `echo "line 2" >> test.txt`
3. Проверь: `cat test.txt`
4. Перезапиши: `echo "new content" > test.txt`
5. Проверь — старое содержимое потеряло?
6. Посчитай процессы: `ps aux | wc -l`
7. Найди свои команды в истории: `history | grep "ls"`
8. Найди 5 самых больших директорий: `du -sh /var/* | sort -rh | head -5`

### Упражнение 3.7: DevOps Think
**Задача:** «Ты подключился к серверу. Найди все конфигурационные файлы Nginx»

Подсказки:
1. Где обычно лежат конфиги Nginx? (`/etc/nginx`)
2. Найди все `.conf` файлы: `find /etc/nginx -name "*.conf"`
3. Сколько их? (`find ... | wc -l`)
4. Покажи содержимое главного конфига: `cat /etc/nginx/nginx.conf`
