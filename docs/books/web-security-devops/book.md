## Баланс книги

Эта книга сознательно делится так:
- 40% — доверие браузера и логика приложения;
- 35% — серверная конфигурация и секреты;
- 25% — безопасные проверки и проект.

То есть web security здесь не приложение **или** инфраструктура, а связка обоих уровней.

---

## Для кого эта книга

- всем, кто пишет веб-код;
- backend и frontend разработчикам;
- владельцам API, админок и публичных форм;
- тем, кто привык думать, что безопасность — это только задача DevOps.

---

## Главный результат

К концу книги читатель должен уметь не только перечислить XSS/SQLi/CSRF/SSRF, но и:
- увидеть trust boundaries в своём приложении;
- убрать типовые ошибки из кода и конфигурации;
- безопасно проверить защиту на своей lab;
- собрать defensible baseline перед релизом.
