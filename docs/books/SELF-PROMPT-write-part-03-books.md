# Self-prompt: написать книги части 3 по порядку

Ты работаешь в репозитории курса DevOps и должен последовательно написать все книги части 3 по security engineering.

Порядок работы:
1. Ориентируйся на `PART-03-security-architecture.md` и `AGENT-INSTRUCTIONS-part-03-security.md`.
2. Пиши книги по порядку: модули 15 -> 21.
3. Каждая книга должна быть самостоятельной и подробной, даже если местами повторяет ранее сказанное.
4. Формат каждой книги:
   - `book.md`
   - `chapter-00.md` ... `chapter-09.md`
   - `appendix-a.md`
   - `appendix-b.md`
5. В каждой книге обязательно должны быть:
   - стартовая точка;
   - кому полезна книга;
   - threat model;
   - практические защитные шаги;
   - безопасные lab-only проверки;
   - итоговый проект;
   - финальный чеклист.
6. Offensive-материал допустим только как:
   - теория;
   - безопасная демонстрация на своих VM/контейнерах;
   - проверка собственной защиты.
7. Нельзя включать вредоносный код, destructive playbook для чужих систем и материалы, делающие курс offensive-учебником.
8. В каждой книге показывай путь от home lab к VPS, small business и enterprise patterns.
9. В первой книге обязательно дай 3 варианта итогового проекта:
   - один защищённый публичный Linux-сервер;
   - gateway + DMZ + app server;
   - лаборатория со сценариями атак и доказательством защиты.
10. Пиши инженерным русским языком: ясно, конкретно, без воды.

Результат работы: полный первый драфт всех книг части 3, пригодный для чтения в существующем markdown-reader курса.
