# Глава 0: Модель угроз — от чего защищаемся

> **Запомни:** Цель не "стать неуязвимым" (это невозможно), а "стать неинтересной целью". Боты ищут лёгкое — дай им отпор, пойдут дальше.

---

## 0.1 Кто реально атакует маленькие серверы

Не хакеры в чёрных худи из фильмов. Реальные угрозы проще и скучнее.

### Боты-сканеры (24/7)

Автоматические программы которые сканируют весь интернет:

```
Бот: "Порт 22 открыт → пробую root:root, root:123456, admin:admin..."
Бот: "Порт 5432 открыт → пробую стандартные пароли PostgreSQL..."
Бот: "Nginx версии 1.18 → знаю 3 уязвимости, пробую..."
```

Прямо сейчас, пока ты читаешь, боты сканируют SSH-порт твоего сервера.

```bash
# Посмотри на своём сервере:
grep "Failed password" /var/log/auth.log | wc -l
# Скорее всего: десятки или сотни попыток за день
```

### Криптомайнеры

Ищут слабые места и устанавливают майнер:

```
Нашёл дыру → скачал майнер → запустил → твой CPU майнит кому-то криптовалюту
```

Результат: сервер тормозит, ты платишь за электричество.

### Шантажисты (ransomware)

Шифруют данные и требуют выкуп.

```
Нашёл доступ к БД → зашифровал → "плати 0.1 BTC или данные удалены"
```

> **Защита:** бэкапы (Модуль 5). Если данные есть — шантажист бессилен.

## 0.2 Что они ищут конкретно

| Цель | Что ищут |
|------|----------|
| SSH | `root:root`, `root:123456`, `admin:admin` и тысячи других |
| Базы данных | Открытый порт 5432, 3306, 27017 без пароля |
| Веб-сервер | Устаревший Nginx с известными уязвимостями |
| Формы логина | Брутфорс паролей пользователей |
| Файлы | `.env`, `.git/config`, `wp-config.php` — утечка секретов |

### Посмотри сам

```bash
# Сколько неудачных входов сегодня
grep "$(date +%b %e)" /var/log/auth.log | grep "Failed password" | wc -l

# Какие IP чаще всего пытаются
grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -rn | head -5

# Что пробуют
grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -rn | head -5
```

## 0.3 Чего обычно НЕ бывает

| Не бывает | Почему |
|-----------|--------|
| Целенаправленных атак | Ты не банк и не правительство |
| APT-группировок | Им нужны крупные цели |
| Нулевых дней (zero-day) | Слишком дорого для маленького сервера |

> **Запомни:** Большинство атак — автоматические.
> Бот пробует стандартные пароли → не получилось → идёт дальше.
> Твоя задача: не быть лёгкой целью.

## 0.4 Модель защиты этого курса

```
Слой 1: Закрыть очевидные дыры
  → SSH только по ключам, пароли отключены

Слой 2: Замедлить автоматические атаки
  → fail2ban банит после 5 неудачных попыток

Слой 3: Сократить поверхность атаки
  → Закрыты все порты кроме 22, 80, 443
  → Отключены ненужные сервисы

Слой 4: Узнавать о попытках
  → Логи + алерты в Telegram
  → fail2ban логи: "IP заблокирован"

Слой 5: Быстро восстанавливаться
  → Бэкапы (Модуль 5)
  → Runbook для аварий (Глава 8)
```

### Визуально

```
Интернет
  │
  ├── [Слой 1: SSH ключи] ─── пароли не работают
  │
  ├── [Слой 2: fail2ban] ─── бан после 5 попыток
  │
  ├── [Слой 3: ufw] ─── только 3 порта открыты
  │
  ├── [Слой 4: мониторинг] ─── ты узнаёшь об атаке
  │
  └── [Слой 5: бэкапы] ─── даже если всё сломалось
```

Каждый слой — это отдельная глава книги.
