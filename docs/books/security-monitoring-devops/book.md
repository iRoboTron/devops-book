## Что читатель настроит к концу книги

```
Безопасность:
├── SSH: только ключи, пароли отключены
├── fail2ban: блокировка брутфорса (SSH, Nginx)
├── ufw: открыты только нужные порты
├── автообновления: unattended-upgrades
└── lynis: аудит безопасности

Мониторинг:
├── Netdata: дашборд CPU/RAM/диск/сеть
├── health-monitor.sh: алерты в Telegram
├── daily-report.sh: ежедневный отчёт
└── logwatch: дайджест логов

Результат:
└── Telegram получает:
    ├── "Сервис упал"
    ├── "Диск 85%"
    ├── "IP заблокирован fail2ban"
    └── "Ежедневный отчёт: всё в порядке"
```

---

## Главная идея

**Безопасность — это слои.** Нет одной настройки которая защищает всё. Каждый слой усложняет жизнь злоумышленнику.

**Мониторинг — это уши и глаза.** Ты не можешь смотреть в терминал 24/7. Система смотрит за тебя и говорит "здесь проблема" раньше катастрофы.

## Как пользоваться книгой

1. **Читай по порядку** — главы строятся друг на друге
2. **Делай упражнения** — особенно Telegram-алерты (обязательно!)
3. **Отмечай чеклисты** — в конце каждой главы
4. **Напиши runbook** — финальная глава, твоя инструкция на 3 ночи

## Предварительные требования

- Пройдены Модули 1–5
- Работающий сервер с Docker, Nginx, PostgreSQL
